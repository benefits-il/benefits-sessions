# In Class — Meta-Agents

## ✏️ Feed in the methodology, challenge it, and run the research

Now, in class: you take SPOKE™/SPOKEN™, tell Claude it is not good enough, and send a research plan to several tools at once, so you can learn how the world really builds agents.

## The instructions

1. Feed Claude your SPOKE/SPOKEN methodology, then challenge it: this methodology is not good enough; what are the proven ways the world builds agents? After that, ask your DR Skill to turn it into a research plan. Paste the following prompt:

```
I have a method for building agents that I call SPOKE / SPOKEN:
S (Self) — who the agent is: role, audience, character.
P (Principles) — three to five working principles: what the agent always does and what it never does.
O (Outputs) — what the agent receives as input and what it returns as output, and in what format.
K (Knowledge) — what the agent must know about my world: documents, terms, examples, preferences.
E (Execution) — how the agent works step by step, in a linear sequence.
N (Next) — the component that turns SPOKE into SPOKEN: it declares that the agent is one link in a chain, who the next link is, and what it receives as input.

I do not think this method is good enough. I am sure that out in the world people
build agents in proven and better ways. I want to know what the
proven methods for building agents in the world are — the form in which agents are built
so that they always work, always function, always carry out the tasks, by
a correct structure suited to AI.

The agents I am going to build will run across various platforms — Claude Co-work,
Cursor, Claude Code, and cloud agent platforms. I want the methods you find to suit
any AI platform that can run agents — both the structure and the agent's content.
No limit on content: the agents should not be too short, but thick agents.

I want to run Deep Research on this using my DR Skill.
Write me an organized research plan that I will run across several tools in parallel, so I get
a full answer on the most correct ways to build an agent.
```

2. Take the resulting plan and paste it into several of your favorite DR tools, in parallel. Run each one, then leave them be. Do not sit and watch the screen.

3. As the results start coming back over the next few minutes, save each one into a single folder on your computer.

## The desired result

A research plan on how the world builds agents, run in parallel across several DR tools in the background, with the results gathered into one folder, ready to become Banay at home.

## Challenge (optional)

If you have a fourth or fifth DR tool, add it to the run: more independent points of view, one merge later on.
