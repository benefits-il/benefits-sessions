## בבית: לתת ל-Benefit את כל מה שנשאר

בכיתה בנינו את Benefit ונתנו לו שני סקילים, שמירה ושליפה. הוא כבר עובד. עכשיו, בבית, משלימים את התמונה: את שאר הסקילים, את כל הידע שצברנו עד היום, וכמה הרצות אמיתיות שהופכות אותו משלכם-על-הנייר לשלכם-באמת. ואם לא הספקתם לסיים משהו בכיתה, כאן המקום להשלים, ולעבור על כל ארבע המשימות עד הסוף, בסדר הזה.

## משימה ראשונה: לסיים את Benefit ולתת לו את הידע

מי שלא הספיק לבנות את Benefit בכיתה, מתחיל מכאן. פותחים פרויקט חדש בקלוד, מדביקים את הסוכן המוכן להוראות הפרויקט, ומחברים את Notion עם הרשאת קריאה וכתיבה. אחר כך מעלים את כל קבצי הידע, וביניהם קובץ מבנה-הקורס. הכל יושב בעמוד אחד: **benefits-il.dev/benefit-agent**.

ומי שכן בנה אותו בכיתה, עדיין שווה לעצור ולוודא שהכל בפנים. הדביקו ל-Benefit את הפרומפט הבא:

```
מי אתה, ואילו קבצי ידע יש לך כרגע? עבור עליהם אחד-אחד, ותגיד לי במיוחד אם יש לך את קובץ מבנה-הקורס (course-structure) ואת חומרי היחידות. אם משהו חסר, תגיד לי בדיוק מה.
```

**למה זה:** סוכן בלי קבצי הידע שלו לא יודע לתייק כלום במקום הנכון, וקובץ מבנה-הקורס הוא בדיוק מה שהופך את התיוק לאוטומטי. בלעדיו Benefit מנחש, ואיתו הוא יודע.

## משימה שנייה: שני הסקילים שנשארו, והרצה אחת לכל אחד

בכיתה התקנתם שמירה ושליפה. נשארו שניים, ושניהם באותו עמוד הורדה. מתקינים כל אחד בקלוד דרך Settings ← Skills ← Upload, ומריצים אותו פעם אחת על חומר אמיתי שלכם.

**הסקיל הראשון, ייעוץ ופירוק לפי STUDIO.** לוקח משימה אמיתית שלכם ומפרק אותה לשש תחנות, עם המלצה באיזה כלי להשתמש בכל שלב. הריצו אותו פעם אחת:

```
יש לי משימה: [המשימה שלכם בשורה אחת]. תפרק לי אותה לפי STUDIO, ותגיד באיזה כלי כדאי להשתמש בכל שלב.
```

**הסקיל השני, בונה פרומפט לפי MICRO.** בונה לכם פרומפט נקי ומסודר, ומחזיר אותו מוכן להעתקה. הריצו אותו פעם אחת:

```
אני צריך פרומפט ש[מה שאתם רוצים שהפרומפט יעשה]. תבנה לי אותו לפי MICRO.
```

**למה זה:** כל סקיל הוא עוד דבר אחד ש-Benefit עושה בשבילכם. ואתם סומכים על סקיל רק אחרי שראיתם אותו עובד על החומר שלכם, לא לפני. הרצה אחת זה כל מה שצריך.

## משימה שלישית: כל פרקי-הלמידה, לתוך נושן

עד עכשיו Benefit מכיר את מה ששמרנו יחד בכיתה. עכשיו נותנים לו את כל מה שלמדנו עד היום, כדי שתהיה לו את התמונה המלאה. שמונה פרקי-למידה, משני השיעורים:

משיעור 1, יסודות, סקילים, סוכנים. ומההכנה לשיעור 2, שפה משותפת, connectors, נושן, שרשור סוכנים, ו-deep research.

פותחים שיחה חדשה עם Benefit. נכנסים לעמוד של כל פרק-למידה באתר, לוחצים על הכפתור "העתק לנושן", ומדביקים את הפרק לשיחה. בראש השיחה, לפני הפרק הראשון, מדביקים את הפרומפט הבא:

```
אני אדביק לך עכשיו פרקי-למידה מהקורס, אחד-אחד. לכל פרק: זהה לאיזו יחידה ולאיזה שיעור הוא שייך, מצא לו את המקום הנכון במחברת (Benefits ← השיעור ← היחידה), והוסף לו את תיוג ה-STUDIO המתאים. לפני שאתה כותב משהו לנושן, הראה לי קודם איפה הפרק הולך ואיזה תיוג נתת, וחכה לאישור שלי. מתחילים. הפרק הראשון:
```

מתחת לפרומפט מדביקים את הפרק הראשון. אחרי ש-Benefit הראה לכם איפה הוא מתייק ואישרתם, כותבים "הפרק הבא:" ומדביקים את הבא, וכך עד ששמונת הפרקים בפנים.

**למה זה:** זה מה שהופך את Benefit מ"מכיר את השיעור של היום" ל"מכיר את כל הקורס". והתיוק לפי המבנה הוא בדיוק מה שיאפשר לו אחר כך לשלוף לכם את העמוד הנכון בלי לחפש בעיניים עצומות.

## משימה רביעית: החלטה אמיתית מהשבוע

בחרו החלטה אחת אמיתית שאתם יושבים עליה השבוע. כלי שצריך לבחור, מייל שקשה לנסח, פרויקט קטן שלא יודעים מאיפה לתפוס. תנו אותה ל-Benefit, בקשו ייעוץ לפי STUDIO, ובסוף שמרו את התובנה לנושן:

```
יש לי החלטה אמיתית מהשבוע: [תארו אותה בשתי שורות]. אני רוצה ייעוץ לפי STUDIO: מאיפה להתחיל, באיזה כלי להשתמש בכל שלב, ומה הדבר הראשון שכדאי לי לעשות כבר עכשיו. בסוף, סכם את התובנה המרכזית ושמור אותה לנושן בעמוד התובנות שלי.
```

**למה זה:** זאת כל הנקודה של Benefit. לא לאחסן את הקורס, אלא לשבת לידכם על החלטה אמיתית, ולשמור את מה שעזר לכם לראות, כדי שבפעם הבאה הוא כבר שם.

## כמה דברים שכדאי לזכור

הזיכרון תמיד בידיים שלכם: אפשר לפתוח את Notion ולתקן שם או תאריך בעצמכם. עדיף לתת ל-Benefit לתייק, אבל כשצריך, אתם יכולים. אם פעולת כתיבה לא עוברת, ודאו שחיבור Notion מחובר עם הרשאת כתיבה, ואשרו את כלי הכתיבה כשהוא מבקש. ואם נתקעתם באיזשהו שלב, נסו שוב, ונסדר את זה יחד במפגש.
