# בכיתה — בונים מנוע

## מה בונים כאן

את המושג MCP כבר פגשתם בהכנה. המשפט שחשוב לקחת ממנו אחד: MCP זו הדרך שלכם לשחרר החוצה דברים. ככה שולחים מייל ממותג מתוך קלוד, מעלים סרטון ליוטיוב, שולפים הקלטות מזום, ומסנכרנים מידע ליומן — כל אחד מהחיבורים האלה הוא MCP אחד.

היום בונים את הראשון שלכם: מנוע-מדיה שעוטף את Gemini ונותן לקלוד שש יכולות חדשות — תמלול, ניתוח, תמונות, וידאו, דיבור ומוזיקה. בלי שורת קוד אחת משלכם.

מרגע שתדביקו לקלוד את מסמך-הבנייה, הוא עושה בשבילכם חמישה דברים — קו-ייצור אחד:

1. מתחבר לשירות של Gemini.
2. עוטף אותו בשרת MCP.
3. מגדיר את ששת הכלים.
4. שם את המפתח בקובץ env — לא בצ'אט.
5. בודק שהכול חי.

זה קו-הייצור של כל MCP בעולם. ברגע שראיתם אותו פעם אחת, תזהו אותו בכל חיבור שתבנו בעתיד.

עובדים בשני מסלולים במקביל: מסלול המנוע רץ לבד אחרי ההדבקה, ובזמן שקלוד בונה — מוסרים במסלול השני את סקיל המותג לבנאי. שני המסלולים נפגשים בתמונה אחת על-מותג.

## מסלול המנוע

מורידים מהעמוד הזה את [מסמך הבנייה של המנוע](mcp-engine-build.md), ומתחילים:

1. פותחים תיקייה חדשה וריקה, ופותחים אותה בקלוד קוד.
2. מדביקים לקלוד את מסמך-הבנייה, ומבקשים ממנו:

```text
תבנה לי לפי המסמך הזה
```

או, אותו פרומפט באנגלית:

```text
Build it for me according to this document
```

מרגע ההדבקה קלוד עובד לבד כמה דקות, והמסמך מוביל אותו עד הסוף — הרישום ב-Claude Code, המפתח, בדיקת-העשן ורענון-הסשן.

**המפתח הוא סוד.** את מפתח ה-Gemini מדביקים רק בקובץ ה-env שקלוד יפתח לכם, אף פעם לא בצ'אט. מה שמודבק בצ'אט נשלח ונשמר בשיחה; מה שמודבק בקובץ נשאר במחשב שלכם. אם קלוד מבקש לסגור ולפתוח מחדש את קלוד קוד — עושים את זה, כי המנוע נתפס רק בסשן חדש.

בודקים שהמנוע חי — בדיקה חינמית:

```text
אילו כלים יש לך מהמנוע gemini-engine?
```

או, אותו פרומפט באנגלית:

```text
What tools do you have from the gemini-engine server?
```

אם החיבור חי, קלוד ימנה את ששת הכלים. הבדיקה הזאת לא עולה כלום.

## מסלול הסקיל — בזמן שקלוד בונה

בזמן שקלוד בונה את המנוע, פותחים את החלון השני ומוסרים לבנאי — הסוכן בונה-הסקילים שלכם משיעור 2 — את סקיל המותג. בנאי יבנה לכם ארט-דיירקטור אישי: מעכשיו, כל פעם שתבקשו תמונה, סרטון או קריינות, הסקיל כבר מכיר את העיצוב שלכם ודואג שהתוצאה תצא על-המותג.

מורידים את [מסמך ההוראות של סקיל המותג](brand-skill-instructions.md), ואז:

1. פותחים את קלוד קוד בתיקיית-הפרויקט שלכם — אותה תיקייה שבה יושבים הלוגו, קובץ DESIGN.md וחומרי המותג משיעורים 3 ו-4.
2. מעירים את בנאי, מוסרים לו את המסמך, ומבקשים:

```text
בנאי — תבנה לי את סקיל המותג לפי מסמך ההוראות הזה. תסביר מה הבנת ותחכה לאישור שלי לפני שאתה בונה.
```

או, אותו פרומפט באנגלית:

```text
Banay — build my brand skill from this instructions document. Explain what you understood and wait for my approval before you build.
```

בנאי ישאל אתכם רק על מה שחסר — הלוגו, הצבעים, החומרים — והתשובות כבר יושבות בתיקייה שלכם משיעורים 3 ו-4.

אין לכם עדיין את בנאי בקלוד קוד, או שחומרי המותג לא בתיקייה? ממשיכים עם המנוע — סקיל המותג מצטרף כשהחומרים מוכנים.

## נקודת המפגש — תמונה אחת על-מותג

כאן שני המסלולים נפגשים. כשהמנוע רשום וחי, וכשבנאי סיים לבנות את הסקיל — מבקשים מהסקיל תמונה אחת על-מותג, והוא מפעיל בשבילכם את הכלי generate_image של המנוע:

```text
דרך סקיל המותג — תייצר לי תמונת פתיחה לפוסט על הפרויקט שלי
```

או, אותו פרומפט באנגלית:

```text
Through my brand skill — generate a cover image for a post about my project
```

זו קריאה בתשלום — תמונה אחת עולה בערך שבעה סנט (כ-$0.067) מהקרדיט שטענתם בהכנה. כשהתמונה הזאת מופיעה, עם הצבעים שלכם — יש לכם מנוע, ויש לכם סקיל.

> **נתקעתם?** קודם כול שאלו את קלוד. לכל אחד משני המסמכים יש טבלת "אם נתקעתם" בסוף — שם פתרון לכל תקלה נפוצה: המפתח שנדחה, הכלים שלא מופיעים, או תמונה שיצאה בצבעים קרובים אבל לא מדויקים.

## מה יש לכם ביד עכשיו

- מנוע-מדיה רשום בקלוד קוד, עם שש יכולות — תמלול, ניתוח, תמונות, וידאו, דיבור ומוזיקה. בונים אותו פעם אחת, והוא משרת הכול.
- סקיל מותג אישי שבנאי בנה — ארט-דיירקטור שמכיר את הצבעים, הגופנים והקול שלכם, ויודע להפעיל את המנוע.

שני הנכסים האלה הם התשתית של כל מה שנבנה מכאן והלאה. מכאן בוחרים עולם.
