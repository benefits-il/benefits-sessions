# בכיתה — BMAD

היחידה האחרונה של הקורס, ואין בה מושג חדש ללמוד. את BMAD כבר פגשתם ביחידת ההכנה — הסרטון של בן הסביר את השיטה, וההתקנה כבר יושבת אצלכם. כאן פשוט עובדים איתה.

זו נקודת-השיא של כל מה שבניתם. אתם מגיעים עם חבילה שנאספה לאורך הקורס: קול-המותג והאיפיון משיעור 3, מאגר-הידע ומערכת-העיצוב משיעור 4, והסקיצות של המסכים מההכנה לשיעור הזה. כל זה יושב בתיקיית-הפרויקט שלכם — והצוות של BMAD יונק ממנו כשהוא שואל.

---

## הצוות שעובד בשבילכם

BMAD הוא צוות וירטואלי, ולכל סוכן תפקיד אחד: מרי מזקקת את הרעיון, ג'ון כותב את מסמך-הדרישות, ווינסטון מתכנן את הארכיטקטורה, ואמיליה בונה את הקוד. בין כל שלב יש שער-איכות — נקודת Go/No-Go שבה קוראים את המסמך שנוצר ומאשרים לפני שממשיכים.

וזה כל הסיפור של היחידה: אתם לא כותבים קוד — אתם מנהלים צוות שכותב. מגדירים, קוראים, מאשרים. אתם הבמאי.

---

## הרצף — צעד אחרי צעד

1. **פותחים את תיקיית-הפרויקט בקלוד קוד** — זו עם החבילה בפנים: קול-המותג, האיפיון, מאגר-הידע, מערכת-העיצוב והסקיצות.
2. **מריצים `bmad-help` — בדיקת-חיים.** זה ה-Master: הוא סורק את הפרויקט ואומר לכם מה הצעד הבא. אם הוא עונה — ההתקנה עובדת. נתקעתם אי-פעם בהמשך? חוזרים אליו.
3. **צ'אט נקי, `bmad-product-brief`.** מרי מתחילה לשאול — מה בונים, למי, מה התוצר. השאלות מוכרות: אלה שאלות התנור. ויש לכם יתרון — רוב התשובות כבר כתובות בתיקייה שלכם.
4. **צ'אט נקי לכל פקודה, ובין פקודה לפקודה — קוראים ומאשרים.** כל פקודה רצה בצ'אט נקי משלה. כשמסמך נוצר, פותחים אותו, קוראים בלוק, ומחליטים: מאשרים או מתקנים. זה רגע ה-Go/No-Go — אם המסמך לא מדויק, גם הקוד לא יהיה מדויק.

---

## המפה המלאה

זו כל הדרך, בשורה אחת. אין כאן חלוקה — כל אחד מתקדם לפי הקצב שלו, עד איפה שהוא מגיע:

```text
brief → prd → architecture → epics → readiness → sprint → stories
```

ולכל תחנה — הפקודה, הסוכן, והתוצר:

| פקודה | סוכן | תוצר |
|---|---|---|
| `bmad-help` | BMad Master | המלצה לצעד הבא |
| `bmad-product-brief` | מרי | `product-brief.md` |
| `bmad-prd` | ג'ון | `prd.md`, `decision-log.md` |
| `bmad-ux` (אם יש ממשק) | סאלי | `DESIGN.md`, `EXPERIENCE.md` |
| `bmad-create-architecture` | ווינסטון | ארכיטקטורה |
| `bmad-create-epics-and-stories` | — | epics + stories |
| `bmad-check-implementation-readiness` | ווינסטון | דוח מוכנות |
| `bmad-sprint-planning` | אמיליה | `sprint-status.yaml` |
| `bmad-create-story → bmad-dev-story → bmad-code-review` | אמיליה | קוד + בדיקות |

---

## לשים לב

- כותבים את שם-הפקודה ישירות, בלי slash לפניו. אם ה-RTL הופך לכם את הסימן — פשוט מקלידים `bmad-prd` כמו שהוא.
- מצאתם מדריך ישן עם `/pm` או `/architect`? זה V4/V5. אצלנו V6 — פקודות `bmad-*` ישירות.
- כל פקודה בצ'אט נקי. "סתם לדבר" עם הסוכן במקום להריץ פקודה — זה מה שמבלבל אותו.

## אם נתקעתם

| מה קרה | מה עושים |
|---|---|
| `npx bmad-method install` נכשל | בודקים `node -v` (צריך 20.12+), ומריצים בתיקייה ריקה |
| הסוכן "לא מבין" את הפקודה | צ'אט נקי + שם הסקיל ישירות (`bmad-prd`), בלי slash |
| מצאתם מדריך עם `/pm` או `/architect` | זה V4/V5 ישן; V6 עובד עם `bmad-*` ישיר |
| אין סביבת IDE, או שההתקנה תקועה | Web Bundle ל-ChatGPT/Gemini (bmadcode.com/web-bundles) |
| הצ'אט "שוכח" באמצע | זה ה-drift; פותחים צ'אט נקי לכל פקודה |

## מה הצעד הבא שלך?

בדיקה אחת: כתבו לעצמכם משפט אחד — איזו פקודה תריצו, ועל מה. לא "אמשיך לעבוד", אלא משפט מדויק: `bmad-prd` על הפרויקט שלי, או `bmad-create-architecture` אחרי שאישרתי את ה-PRD. אם אתם יודעים מה הפקודה הבאה שלכם — אתם יודעים לנהל את הצוות הזה.
