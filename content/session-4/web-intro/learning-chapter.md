# 🌐 שלושה קבצים, שלושה שחקנים ורעיון אחד גדול

> פרק למידה · יחידת איך בנוי עמוד אינטרנט. קול Story. נכתב מתחנת המושגים של שיעור 4: המצגת והערות המנחה, בלי תוספות.

## מה תדעו לעשות בסוף הפרק

לזהות את שלושת הקבצים שמרכיבים כל עמוד אינטרנט ולדעת מה התפקיד של כל אחד, להסביר למה הקובץ הראשי נקרא דווקא index.html, למקם את שלושת השחקנים שמאחורי כל אתר ולהבין איפה GitHub Pages נכנס לתמונה, ולענות בביטחון על שתי שאלות: למה React קיימת, ולמה העמוד שלכם לא צריך אותה. ומעל הכול — לצאת עם הרעיון המרכזי של התחנה: העבודה הקשה עברה מהבנייה אל התכנון.

**מה כדאי שיהיה פתוח:** שום דבר. הפרק הזה נקרא, לא מוקלד. אם בא לכם, פתחו לצד הקריאה את הריפוזיטורי של העמוד האישי שהעליתם בבית — תזהו בו את הקבצים תוך כדי.

**על מה זה נשען:** יחידות ההכנה של השיעור — העבודה בטרמינל ועם git, העמוד האישי שחי באינטרנט, וסרטון המוצר שיצרתם.

**מפת הפרק:** נתחיל מהשאלה למה בכלל צריך את השפה הזאת. אחר כך נפגוש את שלושת הקבצים ונציץ להם פנימה, נכיר את שלושת השחקנים שמאחורי כל אתר, נבין למה נולדה React, ונגיע לרעיון הגדול של התחנה. נסגור עם דף עזר מהיר.

---

## למה בכלל צריך את השפה הזאת

בבית קרה משהו ששווה לעצור עליו: העליתם עמוד אמיתי לאינטרנט. הוא חי, יש לו כתובת, אפשר לשלוח אותה לחברים. אבל אם מישהו היה שואל אתכם מה בדיוק יש בפנים — מה הם הקבצים האלה, מי מגיש אותם, ואיך הדפדפן הופך אותם לעמוד — רוב הסיכויים שהייתם מושכים בכתפיים.

וזה בדיוק הפער שהתחנה הזאת סוגרת. קלוד כותב בשבילנו את הקוד, אבל כשהוא מציג לנו קבצים ומסביר מה הוא עשה, אנחנו רוצים להבין מה שייך למה. לא כדי לכתוב בעצמנו — כדי לזהות, לכוון ולבקש נכון. המטרה שלנו היא להכיר ולזהות; את הכתיבה עצמה עושה קלוד.

---

## שלושה קבצים, תפקיד אחד לכל אחד

כל עמוד באינטרנט בנוי משלושה קבצים, וכל קובץ אחראי על דבר אחד:

- קובץ HTML מגדיר את המבנה של העמוד — הכותרות, הפסקאות, התמונות והסרטון.
- קובץ CSS מגדיר את המראה — הצבעים, הגדלים, הפריסה על המסך.
- קובץ JavaScript מוסיף התנהגות — מה קורה כשלוחצים, מה זז ומה מגיב.

כמה דיוקים חשובים על השלושה. HTML היא שפת סימון: תגיות שאומרות לדפדפן מה כל דבר — כותרת, פסקה, תמונה. HTML וגם CSS הם קוד, אבל לא שפת תכנות. JavaScript היא כן שפת תכנות, והיא מה שהופך עמוד סטטי לעמוד שמגיב — הפונקציונליות. ו-CSS עובד ככללים: בוחרים רכיב וקובעים איך הוא ייראה, וכלל אחד יכול לשנות את כל העמוד.

---

## הקבצים מבפנים, ושער הכניסה

בכיתה הצצנו לשלושת הקבצים מבפנים, דרך שלושה כרטיסי קוד קצרים. בקטע ה-HTML רואים תגיות: פותחות וסוגרות, וביניהן התוכן — למשל כותרת ותג וידאו. בקטע ה-CSS רואים כלל: איזה רכיב, ואילו תכונות — צבע, רקע, גודל. בקטע ה-JavaScript רואים הוראה: כשקורה אירוע, תבצע פעולה — למשל, לחיצה שמפעילה את הווידאו.

לא צריך לזכור תחביר. צריך לזהות מה שייך למה כשקלוד מציג לנו קבצים: אם רואים תגיות — זה המבנה; אם רואים כלל של צבעים וגדלים — זה המראה; אם רואים הוראה של אירוע ופעולה — זו ההתנהגות.

ופרט אחד ששווה לשנן: הקובץ הראשי נקרא index.html — זה השם שהשרת מחפש כברירת מחדל, ולכן זה שער הכניסה של כל אתר. גם העמוד שהעליתם בבית נפתח מקובץ בשם הזה.

---

## שלושת השחקנים שמאחורי כל אתר

מאחורי כל אתר עומדים שלושה שחקנים, ולכל אחד תפקיד ברור:

- הדפדפן (Frontend) — מציג את העמוד ומריץ את מה שהמשתמש רואה וחווה.
- השרת (Backend) — מקבל בקשות, מחשב, ומחזיר תשובות. שם יושבת הלוגיקה העסקית.
- מסד הנתונים (Database) — שומר את המידע לאורך זמן: משתמשים, הזמנות, תכנים.

איך זה עובד בפועל? כשגולשים לאתר, הדפדפן מבקש מהשרת את הקבצים, מקבל אותם, ובונה מהם את העמוד אצלכם במחשב. כלומר העמוד לא "נמצא" איפשהו מוכן — הוא נבנה מחדש בדפדפן שלכם, מאותם שלושה קבצים שהכרנו.

ואיפה אתם בתמונה הזאת? העמודים שאנחנו בונים חיים כולם בדפדפן — בלי שרת משלנו ובלי מסד נתונים. GitHub Pages משמש אצלנו כשרת שמגיש קבצים, וזה כל מה שצריך לעמוד סטטי. כשדחפתם את העמוד שלכם בבית, בעצם מסרתם ל-GitHub Pages את הקבצים, והוא זה שמגיש אותם לכל דפדפן שמבקש.

למה בכלל הכרנו את השרת ואת מסד הנתונים אם אנחנו לא משתמשים בהם? כי בשיעור הבא נבנה מערכת שיש בה גם צד שרת וגם מסד נתונים — ואז שלושת השחקנים יהיו על המגרש ביחד.

---

## למה נולדה React

אתרים כמו פייסבוק ויוטיוב מתעדכנים בזמן אמת, בלי לטעון עמוד מחדש. פעם כל לחיצה טענה עמוד חדש; הגישה של רכיבים שינתה את חוויית השימוש באינטרנט. בדיוק בשביל זה נבנו React ו-Angular: העמוד מורכב מרכיבים, וכל רכיב מתעדכן לבד. React פותחה בפייסבוק ו-Angular בגוגל, כדי לנהל ממשקים גדולים ומתעדכנים.

אבל שימו לב לצד השני של המטבע. לעמוד בודד כמו שלנו זה יותר ממה שצריך; לפרויקטים גדולים זה הכלי הנכון. או במילים של בן מהכיתה: «אני אגיד לכם משהו: אני עובד המון עם עמודי HTML.» עמוד אחד עם שלושה קבצים הוא כלי עבודה מצוין, ולא צריך להתבייש בו.

בשיעור הבא נעבוד עם React בעצמנו — קלוד יכתוב את הקוד, ואנחנו נבין מה הוא כותב ולמה.

---

## הרעיון הגדול — העבודה הקשה עברה מהבנייה אל התכנון

זה הרעיון המרכזי של התחנה, וכדאי לתת לו לנשום.

פעם פרויקט אינטרנט התחיל בחודשים של אפיון, עיצוב ופיתוח אצל אנשי מקצוע שונים. תכנון ארוך, הרבה בעלי מקצוע, והכול נשען על ניסיון. גם כשהמידע היה זמין באינטרנט, אדם יצירתי אחד לא יכול היה לעשות את הכול.

היום אפשר לבנות בצורה שיטתית את מה שרוצים, בלי להקים צוות מתכנתים. הבנייה עצמה מהירה וזולה — ולכן הערך עבר למי שיודע לתכנן ולהגדיר מה בדיוק לבנות. מה שמייחד תוצר היום הוא הפונקציונליות, המקוריות והתכנון — לא עצם היכולת לבנות.

ואם זה נשמע מוכר, זה לא במקרה: זה בדיוק מה שאנחנו מתרגלים לאורך כל הקורס. תכנון וסידור של מידע, כדי שקלוד יבצע בצורה הכי טובה שיש.

ומה אפשר לבנות עם זה? עמוד אינטרנט יכול להיות כמעט כל דבר: מצגת, הצעת מחיר, הזמנה לאירוע, דשבורד נתונים, ואפילו משחק. באתר הדוגמאות שראינו בכיתה כל עמוד שונה לגמרי מהאחרים בעיצוב, במטרה ובפונקציונליות — וליד כל עמוד מופיע הפרומפט שיצר אותו, להעתקה ולשימוש בבית.

---

## מחברים הכול

אז מה היה לנו. כל עמוד באינטרנט בנוי משלושה קבצים, וכל קובץ אחראי על דבר אחד: HTML על המבנה, CSS על המראה, JavaScript על ההתנהגות. הקובץ הראשי נקרא index.html, וזה שער הכניסה של כל אתר. מאחורי כל אתר שלושה שחקנים — דפדפן, שרת ומסד נתונים — והעמודים שלנו חיים כולם בדפדפן, עם GitHub Pages בתפקיד השרת שמגיש קבצים. React נולדה בשביל אתרים גדולים שבנויים מרכיבים, ועמוד בודד לא צריך אותה. והרעיון הגדול מעל הכול: העבודה הקשה עברה מהבנייה אל התכנון, ולכן הערך שלכם הוא בתכנון ובמקוריות.

מהיום המילים האלה הן כלי עבודה, לא ז'רגון. יש לנו שפה משותפת — ואיתה עוברים לעבודה.

## דף-עזר מהיר

| המושג | התפקיד |
|---|---|
| HTML | המבנה של העמוד — כותרות, פסקאות, תמונות וסרטון; שפת סימון, לא שפת תכנות |
| CSS | המראה — צבעים, גדלים ופריסה; עובד ככללים, וכלל אחד יכול לשנות עמוד שלם |
| JavaScript | ההתנהגות — מה קורה כשלוחצים; שפת התכנות היחידה מבין השלושה |
| index.html | שער הכניסה של כל אתר — השם שהשרת מחפש כברירת מחדל |
| דפדפן (Frontend) | מציג את העמוד ומריץ את מה שהמשתמש רואה וחווה |
| שרת (Backend) | מקבל בקשות, מחשב ומחזיר תשובות; שם הלוגיקה העסקית |
| מסד נתונים (Database) | שומר את המידע לאורך זמן — משתמשים, הזמנות, תכנים |
| GitHub Pages | השרת שלנו — מגיש קבצים, וזה כל מה שעמוד סטטי צריך |
| React | גישת הרכיבים לאתרים גדולים ומתעדכנים; עמוד בודד לא צריך אותה |

**הכלל:** המטרה שלנו היא להכיר ולזהות. את הכתיבה עצמה עושה קלוד — והערך שלנו נמצא בתכנון ובמקוריות.

**לאן ממשיכים:** מהמצגת עברנו בכיתה להיכרות מעמיקה עם סביבת העבודה, ומשם לבניית עמוד המוצר — שם שלושת הקבצים הפסיקו להיות מושגים והפכו לעמוד אמיתי עם הסרטון שלכם בראשו. ובשיעור הבא נפגוש את React ואת צד השרת בעצמנו.
