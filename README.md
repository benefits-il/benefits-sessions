# benefits-sessions — אתר המסירה של הקורס

אתר HTML סטטי, ממותג בשפת Benefits, שאליו שולחים את התלמידים בלינק. שם הם צופים בסרטונים, קוראים פרקי למידה, ועושים תרגילים ושיעורי בית. זהו אתר הביניים עד שהפלטפורמה (benefits-il) מוכנה.

**חי בכתובת:** https://benefits-il.dev/benefits-sessions/

## איך זה בנוי

שני חלקים:

1. **מנוע** — `viewer.html`: מרנדר כל קובץ Markdown לעמוד מעוצב, דרך `viewer.html?doc=<נתיב>`. תומך בכל מה ש-Markdown יודע (טבלאות, דיאגרמות Mermaid, קוד מודגש עם כפתור העתקה, תמונות, RTL/LTR אוטומטי), וקורא frontmatter כדי לבנות כותרת ממותגת, סלוט סרטון, הורדות וניווט.
2. **קליפת ניווט** — עמודי HTML סטטיים קטנים: שער (`index.html`), אינדקס שיעורים (`sessions.html`), עמוד מפגש (`session-<N>.html`), ועמוד יחידה (`content/.../index.html`).

הכל נשען על `delivery.css` — אותם טוקנים ואותה שפת רכיבים כמו הפלטפורמה, בלי ה-app shell. כשהפלטפורמה תהיה מוכנה, ההגירה טריוויאלית.

## מפת הניווט

שער ← אינדקס שיעורים ← עמוד מפגש (הכנה · בכיתה · בבית) ← עמוד יחידה (hub) ← עמוד תרגול · עמוד פרק למידה. חזרה-אחורה ו-breadcrumb בכל רמה.

## להוסיף ולעדכן תוכן

- **[HOW-TO.md](HOW-TO.md)** — המדריך המעשי: איך מעדכנים תוכן קיים, מוסיפים סרטון, מוסיפים יחידה או מפגש, ואיך דוחפים live. **התחל כאן.**
- **[CONTENT-CONTRACT.md](CONTENT-CONTRACT.md)** — פירוט ה-frontmatter המלא לכותבי תוכן.
- **[CLAUDE.md](CLAUDE.md)** — סוכן התחזוקה (אם עובדים עם Claude Code).

## הרצה מקומית

הדפדפן חוסם טעינת קבצים מקומיים ב-`file://`. כדי לראות הכל בלחיצה אחת, הריצו שרת מקומי מתיקיית הריפו:

```
npx serve
```

או `python -m http.server`, ואז פתחו את הכתובת המקומית. ב-GitHub Pages הכל עובד מעצמו.

## מבנה הקבצים

- `index.html` · `sessions.html` · `session-2.html` — קליפת הניווט.
- `viewer.html` — המנוע.
- `delivery.css` — שכבת העיצוב (טוקנים + רכיבים).
- `content/session-<N>/<unit>/` — תוכן היחידות: `practice.md`, `learning-chapter.md`, `index.html` (hub), וקבצים להורדה.
- `CONTENT-CONTRACT.md` · `CLAUDE.md` — לכותבי תוכן ולסוכן התחזוקה.
