## השיטה — STUDIO
פרויקט הוא **תהליך**, לא משימה. במקום לבנות ישר, עוברים שש תחנות שכל אחת עונה על שאלה אחת, וכל מה שמתברר נרשם בתנור — המסמך המרכזי שמשקף את הפרויקט כולו בכל רגע.

**שש התחנות:**
1. **Scout** — מה עוד לא ידוע? פירוק החלום, סימון פערים, שליחת שאלות מחקר, חיפוש כלים קיימים.
2. **Target** — מה באמת רוצים? חפירה מתחת לרעיון עד לאקסיומה: "לא יכול להיות ש… / אני חייב ש…" — הערך שפוסל כל פתרון שסותר אותו.
3. **Unify** — יחידות התפקוד. חיתוך לפי "דבר אחד ברור", ולכל יחידה עובד: בן-אדם / סוכן / סקיל.
4. **Design** — שפה אחת לכל התוצרים, מבוססת רפרנסים. אסור שם-תואר בלי רפרנס.
5. **Index** — מה מקבלת ← מה מחזירה כל יחידה, סדר הבנייה, ונקודות עצירה בחיבורים.
6. **Operate** — פער קטלני נבדק קודם בקטן (היתכנות); אחרת בונים גרסה רזה מקצה-לקצה (ריצת ניסיון), ורק אז מרחיבים.

**כללי הדרך:** חשיבה לחוד וביצוע לחוד (מזקקים הוראה נקייה לחלון חדש) ← נשבר? מתקנים את המפעל, משנים משתנה אחד ← יחידה גדולה מדי מקבלת מיני-סטודיו.

## התפקיד שלי מכאן
מלווה הפרויקט. בכל חזרה שלך: קורא קודם את התנור ← אומר איפה אנחנו (מלא / חסר / הצעד הבא) ← מציע את התחנה הבאה. ואחרי כל עבודה — מעדכן את התנור שיישאר נאמן למצב.

## מה שאתה ביקשת ממני עכשיו (ואבצע אחרי אישור)
1. ליצור את שש תת-התיקיות: `Scout`, `Target`, `Unify`, `Design`, `Index`, `Operate` — ולרשום ב"מפת התיקייה" מה תפקיד כל אחת.
2. לשאול אותך איפה הפרויקט עומד — יש רעיון? נעשה מחקר? — ולמלא יחד את מה שכבר ידוע.
3. לומר לך את המצב: מה מלא, מה חסר, ומה הצעד הבא לפי השיטה.

**נקודה אחת ששמתי לב אליה:** התנור עדיין ריק לגמרי — גם שם הפרויקט הוא placeholder (`[שם הפרויקט]`), וגם שער הכניסה ריק. אז אחרי שאיצור את התיקיות, השאלה הראשונה תהיה בעצם שער הכניסה: מה הפרויקט, והאם זה אירוע חד-פעמי או דפוס חוזר.

מאשר שאיצור את התיקיות ואתחיל? או שתעדיף שנמלא קודם ביחד את שער הכניסה בעל-פה ורק אז אצור הכל?

## Claude · Opus 4.8

הבנתי. זה בדיוק פרויקט בשביל STUDIO — כי זה **דפוס חוזר** (פוסט בשבוע), לא אירוע חד-פעמי. אני מתרגם את מה שאמרת לשפת התנור, ועוצר לאישור.

## החלום (במשפט)
מכונה שאני מזין לה רעיון גולמי הקשור ל-**TUESDAY**, והיא מוציאה לי אחת לשבוע פוסט קצר, שימושי ומלמד על ניהול עסק לעצמאים — שגורם לקורא לרצות להשתמש בטיוזדיי — כולל תמונה מגניבה בשפת המותג, מוכן לאינסטגרם וללינקדאין.

## מה שכבר ברור לי מהתיאור שלך
- **הקלט:** רעיון גולמי ממני (זרע), לא פוסט מוכן.
- **הפלט השבועי:** פוסט + תמונה תואמת-מותג, בשתי גרסאות — אינסטגרם (ויזואלי, קצר) ולינקדאין (ערכי-מקצועי).
- **הזהות של הפוסט:** לא פרסומת. קודם ערך אמיתי לעצמאי ← ואז טיוזדיי כפתרון טבעי בסוף. זה כנראה הולך להיות ה**אקסיומה** של הפרויקט (תחנת Target): "לא יכול להיות שהפוסט ירגיש כמו מודעה — הוא חייב ללמד משהו שימושי גם למי שלא ייגע בטיוזדיי לעולם."
- **צריך אינטרנט בשני מקומות:** (א) רעיונות/זוויות תוכן לעצמאים, (ב) איך כותבים פוסט קצר שעובד ב-IG וב-LinkedIn. שניהם נכנסים לתחנת **Scout**.
- **שפה חזותית קבועה** = תחנת **Design**, מבוססת רפרנסים של טיוזדיי (לא "מגניב" מופשט — רפרנס ממשי).
- **המידע על החברה בתיקייה** — צריך להעתיק לכאן רק את מה שרלוונטי לכתיבת פוסטים (מה טיוזדיי עושה, קהל היעד, טון, מותג/צבעים). זה מזין את Scout ואת Design.

## איך אני רואה את זה מתפרק ליחידות (תחנת Unify — טיוטה מוקדמת)
1. **קליטת הזרע** — לוקח את הרעיון הגולמי ומחדד לזווית תוכן אחת חדה.
2. **מחקר** — סוכן שמביא רעיונות/עובדות תומכות מהאינטרנט.
3. **כתיבה** — סוכן שכותב את שתי הגרסאות (IG + LinkedIn) בטון של טיוזדיי.
4. **תמונה** — סקיל שמייצר ויזואל בשפת המותג.
5. **הרכבה** — מאחד לפוסט מוכן להעתקה.

(זו רק טיוטה כדי שנדע לאן חותרים — נחדד אותה בתחנה 3.)

## מה שאני צריך ממך כדי להתחיל (שער הכניסה)
1. **איפה התיקייה עם המידע על טיוזדיי?** תן לי נתיב מלא, ואני סורק ומעתיק לכאן רק את הרלוונטי.
2. **טיוזדיי במשפט אחד** — מה המוצר/השירות בעצם נותן לעצמאי? (כדי שאדע מה ה"פתרון" שהפוסטים מובילים אליו.)
3. **תדירות/כמות:** פוסט אחד בשבוע = פריט אחד בכל הרצה, נכון? או שאתה רוצה שכל הרצה תיתן כמה זוויות לבחור מהן?

## מה אני מציע לעשות ברגע שתאשר
- ליצור את שש תת-התיקיות + לרשום את מפת התיקייה.
- למלא בתנור: שער הכניסה, החלום, וטיוטת האקסיומה.
- לסרוק את תיקיית טיוזדיי ולהעתיק לכאן את הרלוונטי (בסיס ל-Scout ו-Design).

מאשר שאצא לדרך? ותן לי בבקשה את נתיב התיקייה של טיוזדיי — זה הדבר הראשון שאני צריך.
