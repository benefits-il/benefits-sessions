# תרגול — שרשור סוכנים

## לפני שמתחילים

בסרטון פגשתם את הצמד הזה: סוכן אחד מסכם פגישה, והסוכן שאחריו מחלץ ממנה את המשימות. עכשיו תורכם לבנות אותו בעצמכם, צעד אחרי צעד. הראשון מכין תוצר ביניים (הסיכום), והשני הופך אותו לתוצר הסופי (רשימת המשימות).

כל פרומפט כאן מוכן להעתקה, ולידו כתוב בדיוק מה להעתיק, לאן, ובאיזו שיחה. אתם לא כותבים כלום מהראש, רק מעתיקים.

מה שכדאי שיהיה פתוח: חשבון Claude. אם כבר בניתם סוכן ביחידת הסוכנים, מצוין. ואם לא, הבלוק הראשון כאן בונה לכם אחד מאפס.

---

## בונוס — תנו לסוכן שם מעוצב

עוד לפני שמתחילים, משהו קטן בשביל הכיף. בניתי לכם כלי חמוד שנותן לסוכן שם מעוצב: רושמים שם (למשל Benefits), בוחרים אימוג'י שבא לכם, מוסיפים קישוטים, ומעתיקים את התוצאה ישר לשם של הסוכן. לגמרי רשות. רק כי נחמד שלסוכן שלכם יהיה שם משלו.

[סגננו שם לסוכן](https://benefits-il.dev/agent-names/)

## חלק א — בונים את הצמד צעד אחרי צעד

הצמד שנבנה: **סוכן מסכם-פגישה ← סוכן מחלץ-משימות.** הראשון מקבל תמלול ומחזיר סיכום, השני מקבל את הסיכום ומחזיר רשימת משימות. שימו לב מה בדיוק עובר ביניהם: לא הפגישה המקורית, אלא הסיכום שהראשון הכין. זה הדבק של השרשרת.

### צעד 1 — בונים את החוליה הראשונה (סוכן מסכם-פגישה)

פתחו שיחה חדשה בקלוד. נקרא לה **שיחת-הבנייה**, נחזור אליה בהמשך.

ראשית, הדביקו את אפיון ה-SPOKE המוכן של הסוכן:

```
אני רוצה לבנות סוכן לפי שיטת SPOKE. הנה האפיון:

Self — סוכן שמסכם תמלול או הקלטה של פגישה.
Principles — לא מנהל מעקב, רק מסכם. לא ממציא דברים שלא נאמרו בפגישה.
Outputs — קובץ MD מאורגן ומסודר, עם כותרות ברורות לשלושה חלקים: נושאים שנידונו, החלטות שהתקבלו (מי, מה, ועד מתי), ונקודות פתוחות.
Knowledge — התמלול עצמו, מי המשתתפים, ומה מטרת הפגישה.
Execution — קורא את התמלול, מזהה החלטות ומשימות מובלעות, ומסכם בקצרה.
```

עכשיו מוסיפים את האות N. שימו לב למיקום: היא נכנסת **בסוף האפיון, אחרי E**. זאת האות השישית, ולכן היא יושבת למטה מתחת לכל השאר, לא מעל. זאת האות שהופכת SPOKE ל-SPOKEN, ומצהירה שהסוכן הוא חוליה בשרשרת ומי בא אחריו. הוסיפו את הבלוק הזה בסוף, מתחת לחמש האותיות:

```
N — הבא בתור בשרשרת:
התוצר שלי הוא תוצר ביניים, לא הסוף.
החוליה הבאה היא סוכן מחלץ-משימות. הוא מקבל את הסיכום שלי כקלט,
והפלט שלו הוא רשימת המשימות הסופית.
```

ועכשיו הדביקו את פרומפט ה"הרחב", שהופך את האפיון המלא ל-system prompt מסודר:

```
למעלה פירטתי אפיון של סוכן לפי שיטת SPOKEN. עבור על האפיון, סדר אותו לפי
חלקים הגיוניים, תקן את מה שאמרתי, והרחב אותו ל-system prompt מדויק שמודל AI
יעבוד איתו היטב כסוכן. הקפד לכלול את האות N — מי החוליה הבאה ומה היא מקבלת
כקלט. הסבר מה הבנת וחכה לאישור כדי להמשיך.
```

קיבלתם system prompt מלא של סוכן מסכם-הפגישה. הקימו ממנו סוכן בדיוק כמו ביחידת הסוכנים: Project עם שם ואייקון, וה-system prompt בתוך ה-Instructions.

### צעד 2 — נותנים לחוליה הראשונה עבודה אמיתית

עכשיו ניתן לסוכן חומר אמיתי לעבוד עליו. הורידו את קובץ התמלול: [תמלול פגישה לתרגול](meeting-transcript.md). זאת פגישת הנהלה ארוכה ועמוסה, עם שבעה נושאים, החלטות, אחראים, תאריכים ונקודות פתוחות, בדיוק כמו פגישה אמיתית מהעבודה.

פתחו שיחה חדשה מול הסוכן שהקמתם, והזינו לו את התמלול. אפשר להעלות את הקובץ, או להעתיק את תוכנו לשיחה.

הסוכן מחזיר קובץ MD מאורגן: הנושאים שנידונו, ההחלטות שהתקבלו עם האחראים והתאריכים, והנקודות הפתוחות. עצרו רגע על מה שקרה כאן. נכנס תמלול מבולגן עם המון מידע, ויצא מסמך מסודר וקריא שאפשר לעבוד איתו. זאת העבודה האמיתית של החוליה הראשונה, היא לקחה כאוס והפכה אותו לסדר. שמרו את הסיכום בצד. הוא תוצר הביניים, והקלט של החוליה הבאה.

### צעד 3 — החוליה השנייה בונה את עצמה (סוכן מחלץ-משימות)

חזרו ל**שיחת-הבנייה** מצעד 1, זאת שבה הרחבתם את הסוכן הראשון. הסוכן כבר יודע מה ה-N שלו, אז הוא יודע בדיוק את מי לבנות. הדביקו:

```
עכשיו תבנה את הסוכן הבא בשרשרת, בהתאמה לשלד SPOKEN.
תאר אותו לפי חמשת הרכיבים (Self, Principles, Outputs, Knowledge, Execution),
כשהקלט שלו הוא הסיכום שהפקתי, והפלט שלו הוא רשימת המשימות הסופית.
את הפלט הוא מגיש בשני קבצים: קובץ MD מסודר של המשימות, וגם קובץ PDF
מעוצב ויפה עם פונט Rubik, תאריך יעד לכל משימה, וצבעים נעימים.
הסבר מה הבנת וחכה לאישור כדי להמשיך.
```

קיבלתם אפיון מלא לסוכן השני, מחלץ-המשימות, כשהקלט שלו הוא הסיכום שהראשון מפיק. שימו לב מה עשיתם כאן: לא כתבתם אותו בכלל. כתבתם N, והשרשרת בנתה את עצמה. הקימו גם אותו כסוכן נפרד.

### צעד 4 — מריצים את השרשרת

קחו את הסיכום מצעד 2, והדביקו אותו לסוכן מחלץ-המשימות. הוא מחזיר שני קבצים: רשימת משימות מסודרת ב-MD, וקובץ PDF מעוצב עם המשימות, האחראים ותאריכי היעד, בפונט Rubik ובצבעים נעימים.

עצרו וראו את כל המסלול. התחלתם מתמלול פגישה ארוך ומבולגן, ואחרי שתי חוליות יש לכם ביד מסמך סיכום מסודר ומסמך משימות מעוצב להגשה. שתי חוליות, כל אחת עשתה את העבודה שלה, והתוצר של הראשונה הזין את השנייה. זאת שרשרת שעובדת, על חומר אמיתי ולא על דוגמה.

## חלק ב — בנו צמד משלכם מהמאגר

עכשיו אותו דבר בדיוק, על צמד אחר. צירפנו לכם מאגר של חמישה-עשר צמדי-סוכנים מתחומים שונים. בחרו צמד שמדבר אליכם (או סוכן משלכם), ובנו את שתי החוליות עם אותם בלוקים בדיוק מחלק א: כתבו N בסוף האפיון של החוליה הראשונה, הרחיבו, ובאותה שיחה בקשו לבנות את הבא.

המאגר המלא, חמישה-עשר צמדים: [מאגר צמדי הסוכנים](agent-pairs-menu.md).

שלושה צמדים לדוגמה, אם בא לכם להתחיל בלי לצאת מהעמוד:

**חוקר ← כותב.** הראשון מלקט מקורות ומחזיר תקציר ממצאים עם ציטוטים. השני מקבל את התקציר ומחזיר מאמר ערוך וזורם. מה שעובר ביניהם: תקציר הממצאים.

**אסטרטג קמפיין ← קופירייטר.** הראשון מגדיר קהל ומסר ומחזיר בריף שיווקי. השני מקבל את הבריף ומחזיר את הטקסטים בפועל. מה שעובר ביניהם: הבריף.

**מארגן תוכן ← בונה פלט.** הראשון מקבל חומר גולמי ומחזיר מבנה מצגת: כותרת ונקודות-מפתח לכל שקף. השני מקבל את המבנה הזה ומחזיר מצגת או עמוד מעוצב. מה שעובר ביניהם: מבנה השקפים.

לכל צמד, שורת ה-N של החוליה הראשונה היא אותה תבנית, רק עם תיאור הסוכן הבא:

```
N — הבא בתור בשרשרת:
התוצר שלי הוא תוצר ביניים, לא הסוף.
החוליה הבאה היא [תיאור הסוכן הבא: מה הוא עושה, מה הפלט הסופי שלו],
והיא מקבלת את התוצר שלי כקלט.
```

## אתגר רשות

למי שכבר בנה צמד, שתי דרכים להמשיך. **חוליה שלישית:** בקשו מהסוכן השני לבנות את הבא אחריו, ותקבלו שרשרת באורך שלוש. **צמד נוסף:** בחרו צמד אחר מהמאגר ובנו גם אותו.
