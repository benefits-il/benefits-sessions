# סיכום · Memory Layers™

## מה שלמדנו

1. **AI הוא חסר מצב (Stateless).** מודל שפה אינו נושא רגעים קדימה. כל הודעה היא קריאה חדשה לחלוטין של כל ההיסטוריה. כשנדמה שהוא זוכר, המערכת מזריקה מחדש את השיחה לחלון ההקשר (Context window) בכל פעם.

2. **שש שכבות של זיכרון.** מהקרובה אליכם לרחוקה: היסטוריית שיחה · System prompt (הוראת המערכת) · Memory (זיכרון מובנה) · Knowledge (קובצי ידע) · knowledgebase (מאגר ידע חי, Notion) · RAG כמו NotebookLM (נפגוש בהמשך). מארבע הראשונות הסוכן רק קורא; המאגר החי הוא הראשון שהוא גם כותב אליו.

3. **Notion הוא הזיכרון, קלוד הוא המעבד.** במקום להכריח מנוע חסר-מצב לזכור, שומרים את המצב מחוץ לו. המאגר אינו שוכח, כי הוא מסד נתונים ולא מודל.

4. **Benefit, היועץ האישי שלכם.** סוכן שמכיר את חומרי הקורס השמורים ב-Notion. מתייעצים איתו על עבודה עם AI, והוא שומר ושולף בשבילכם. הליבה: קובץ מבנה-הקורס שבתוך הידע שלו הוא מה שמאפשר לו לתייק כל דבר חדש במקום הנכון, לבד.

5. **שליפה לפי-צורך (just-in-time).** Benefit לא טוען את כל המחברת. הוא ניגש לאינדקס, בוחר את העמוד הנכון, וטוען רק אותו: אינדקס ← בחירה ← טעינה.

6. **הזיכרון בידיים שלכם.** תמיד אפשר לפתוח את Notion ולראות או לתקן. הזיכרון לא נעול בתוך מודל.

## השורה ששווה לזכור

> סוכן שלא מנסה לזכור, ומאגר מידע שלא שוכח.
