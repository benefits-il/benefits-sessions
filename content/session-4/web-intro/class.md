# בכיתה — איך בנוי עמוד אינטרנט

## מה עשינו יחד

פתחנו את השיעור בחיבור למה שהכנתם בבית: עבדתם עם הטרמינל ועם git, העליתם עמוד אישי שחי באינטרנט, ויצרתם סרטון מוצר. שלושת הדברים האלה משמשים אותנו לאורך כל השיעור — העמוד שהעליתם בבית הוא לא תרגיל, הוא מקבל היום עיצוב חדש, והסרטון שיצרתם נכנס לעמוד המוצר שנבנה יחד.

התחנה הזאת הייתה מצגת מושגים, בלי תרגיל. עברנו יחד על איך בנוי עמוד אינטרנט: שלושת הקבצים ותפקידיהם, שלושת השחקנים שמאחורי כל אתר, למה נולדה React, ומה השתנה בעידן הנוכחי. העמוד הזה שומר את המושגים לחזרה בבית, והמצגת המלאה זמינה בקישור שלמעלה.

---

## שלושת הקבצים — מבנה, מראה, התנהגות

כל עמוד באינטרנט בנוי משלושה קבצים, וכל קובץ אחראי על דבר אחד:

- קובץ HTML מגדיר את המבנה של העמוד — הכותרות, הפסקאות, התמונות והסרטון.
- קובץ CSS מגדיר את המראה — הצבעים, הגדלים, הפריסה על המסך.
- קובץ JavaScript מוסיף התנהגות — מה קורה כשלוחצים, מה זז ומה מגיב.

דייקנו את ההבחנה: HTML היא שפת סימון — תגיות שאומרות לדפדפן מה כל דבר: כותרת, פסקה, תמונה. HTML וגם CSS הם קוד, אבל לא שפת תכנות. JavaScript היא כן שפת תכנות, והיא מה שהופך עמוד סטטי לעמוד שמגיב. CSS עובד ככללים: בוחרים רכיב וקובעים איך הוא ייראה, וכלל אחד יכול לשנות את כל העמוד.

הסתכלנו גם על הקבצים מבפנים: בקטע ה-HTML רואים תגיות פותחות וסוגרות וביניהן התוכן, בקטע ה-CSS רואים כלל שקובע לרכיב צבע, רקע וגודל, ובקטע ה-JavaScript רואים הוראה — כשקורה אירוע, תבצע פעולה. לא צריך לזכור תחביר. צריך לזהות מה שייך למה כשקלוד מציג לנו קבצים — המטרה שלנו היא להכיר ולזהות, ואת הכתיבה עצמה עושה קלוד.

ופרט אחד ששווה לשנן: הקובץ הראשי נקרא index.html — זה השם שהשרת מחפש כברירת מחדל, ולכן זה שער הכניסה של כל אתר.

---

## שלושת השחקנים — דפדפן, שרת ומסד נתונים

מאחורי כל אתר עומדים שלושה שחקנים:

- הדפדפן (Frontend) — מציג את העמוד ומריץ את מה שהמשתמש רואה וחווה.
- השרת (Backend) — מקבל בקשות, מחשב, ומחזיר תשובות. שם יושבת הלוגיקה העסקית.
- מסד הנתונים (Database) — שומר את המידע לאורך זמן: משתמשים, הזמנות, תכנים.

כשגולשים לאתר, הדפדפן מבקש מהשרת את הקבצים, מקבל אותם, ובונה מהם את העמוד אצלכם במחשב. העמודים שאנחנו בונים היום חיים כולם בדפדפן — בלי שרת משלנו ובלי מסד נתונים. GitHub Pages משמש אצלנו כשרת שמגיש קבצים, וזה כל מה שצריך לעמוד סטטי. בשיעור הבא נבנה מערכת שיש בה גם צד שרת וגם מסד נתונים — לכן הכרנו את שלושת השחקנים כבר עכשיו.

---

## React — למה היא נולדה, ולמה עמוד בודד לא צריך אותה

אתרים כמו פייסבוק ויוטיוב מתעדכנים בזמן אמת, בלי לטעון עמוד מחדש. בדיוק בשביל זה נבנו React ו-Angular: העמוד מורכב מרכיבים, וכל רכיב מתעדכן לבד. React פותחה בפייסבוק ו-Angular בגוגל, כדי לנהל ממשקים גדולים ומתעדכנים. פעם כל לחיצה טענה עמוד חדש — הגישה של רכיבים שינתה את חוויית השימוש באינטרנט.

לעמוד בודד כמו שלנו זה יותר ממה שצריך; לפרויקטים גדולים זה הכלי הנכון. בשיעור הבא נעבוד עם React בעצמנו — קלוד יכתוב את הקוד, ואנחנו נבין מה הוא כותב ולמה.

---

## מה השתנה — העבודה הקשה עברה מהבנייה אל התכנון

פעם פרויקט אינטרנט התחיל בחודשים של אפיון, עיצוב ופיתוח אצל אנשי מקצוע שונים, והכול נשען על ניסיון. גם כשהמידע היה זמין באינטרנט, אדם יצירתי אחד לא יכול היה לעשות את הכול. היום אפשר לבנות בצורה שיטתית את מה שרוצים, בלי להקים צוות מתכנתים — הבנייה עצמה מהירה וזולה.

ולכן מה שמייחד תוצר היום הוא הפונקציונליות, המקוריות והתכנון — לא עצם היכולת לבנות. זה בדיוק מה שאנחנו מתרגלים לאורך כל הקורס: תכנון וסידור של מידע, כדי שקלוד יבצע בצורה הכי טובה שיש.

---

## מה עמוד אינטרנט יכול להיות

עמוד אינטרנט יכול להיות כמעט כל דבר: מצגת, הצעת מחיר, הזמנה לאירוע, דשבורד נתונים, ואפילו משחק. סיירנו יחד באתר הדוגמאות וראינו עמודים ששונים לגמרי זה מזה בעיצוב, במטרה ובפונקציונליות — עתידני, קלאסי, לילדים, יוקרתי. ליד כל עמוד באתר מופיע הפרומפט שיצר אותו, ואפשר להעתיק ולהשתמש בבית.

---

## טבלת המושגים

| המושג | התפקיד |
|---|---|
| HTML | המבנה של העמוד — כותרות, פסקאות, תמונות וסרטון |
| CSS | המראה — צבעים, גדלים ופריסה; עובד ככללים |
| JavaScript | ההתנהגות — מה קורה כשלוחצים; שפת התכנות היחידה מבין השלושה |
| index.html | שער הכניסה של כל אתר — השם שהשרת מחפש כברירת מחדל |
| דפדפן (Frontend) | מציג את העמוד ומריץ את מה שהמשתמש רואה וחווה |
| שרת (Backend) | מקבל בקשות, מחשב ומחזיר תשובות |
| מסד נתונים (Database) | שומר את המידע לאורך זמן |
| GitHub Pages | השרת שלנו — מגיש קבצים, וזה כל מה שעמוד סטטי צריך |
| React | גישת הרכיבים לאתרים גדולים ומתעדכנים; עמוד בודד לא צריך אותה |

מהיום המילים האלה הן כלי עבודה, לא ז'רגון. מכאן עברנו למסך המלא של סביבת העבודה, ומשם לבניית עמוד המוצר.
