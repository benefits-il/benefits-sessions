# בכיתה — מצגות

## מה עשינו יחד

לקחנו חברה אחת בשם TUESDAY ושלושה צרכים אמיתיים שיש לה, ובנינו לכל אחד מהם מצגת — בלי להתחיל מדף ריק ובלי להיתקע בעיצוב. גילינו שמצגת היא בסך הכול תוצר לוואי של עבודה טובה על התוכן, ושקיימת דרך מסודרת מאוד לעשות את זה: בעזרת שני סוכנים שמכירים היטב אחד את השני. Point מסדר את התוכן, שזה הקושי האמיתי שלנו, בעוד ש-POWER מלביש את העיצוב, שהוא החלק הקל.

ראינו איך אותו תוכן בדיוק נשלח לשלושה יעדים שונים: למצגת PowerPoint, למצגת Google Slides דרך Gemini, ולעמוד HTML חי ונושם. זו הייתה בדיוק אותה עבודה על התוכן, רק שהיעד השתנה בכל פעם. וזו בעצם הנקודה כולה: התוכן הוא אוניברסלי, והבנייה היא רק הזנב.

מה שאתם צריכים שיהיה פתוח מולכם כדי לעבוד: שני הסוכנים שהתקנתם בבית, Point ו-POWER, יחד עם תיקיית החומרים של מצגת ה-BI שהורדתם. אם חסר לכם משהו, זה הזמן לחזור לרגע להוראות ההתקנה.

---

## התרגול שלכם — מצגת ה-BI

יש לפניכם תיקיית עבודה של מחלקת ה-BI בחברת TUESDAY. המצגת צריכה להכריע בשאלה אחת ברורה: האם משתלם להמשיך לפתח את המוצר, ובפרט את הפיצ'ר "משימות 2.0", או שעדיף לעצור הכול. הנתונים בתיקייה הם בכוונה לא חד-משמעיים. יש כאן ויכוח אמיתי, ואתם אלו שתכריעו אותו על בסיס המספרים בלבד.

יש לנו פה שני שלבים ושני פרומפטים. קודם כל Point בונה את המבנה, ואחר כך POWER מפיק את המצגת עצמה.

**לפני הכול:** צרפו לשיחה את כל הקבצים שנמצאים בתיקיית ה-BI. חלק מהם רלוונטיים וחלקם הם סתם רעש רקע, וזה נעשה בכוונה. תנו ל-Point לעשות את הסינון.

### שלב 1 — Point בונה את מבנה המצגת

הדביקו את הפרומפט הזה מיד אחרי שצירפתם את הקבצים:

```
אתה מחלקת ה-BI של חברת TUESDAY, חברת תוכנה לניהול עבודה לעצמאי ולעסק של איש אחד.

המטרה: לבנות מצגת ניתוח שוק של מחלקת ה-BI שמכריעה החלטה אחת — האם משתלם להמשיך לפתח את המוצר, ובפרט להמשיך להשקיע בפיצ'ר "משימות 2.0", או לעצור.

צירפתי תיקיית עבודה אמיתית ומבולגנת. יש בה מסמכים מסוגים שונים. חלקם רלוונטיים להחלטה וחלקם רעש שלא קשור אליה. המשימה הראשונה שלך היא לסנן: עבור על הכול, החלט מה רלוונטי ומה לזרוק, ואמור לי בקצרה מה כללת ומה השארת בחוץ ולמה.

אחרי שסיננת, אל תקפוץ ישר למצגת. רואיין אותי קודם. שאל אותי שאלות ממוקדות שיעזרו לחדד את המצגת: מי הקהל, כמה שקפים, איזו עמדה המצגת אמורה להוביל אם הנתונים מצדיקים אותה, ומה רמת הפירוט. שאלה אחת בכל פעם.

הנתונים לא חד-משמעיים. חלק מהאותות תומכים בהמשך הפיתוח וחלק תומכים בעצירה. אל תייפה ואל תכריע מראש. המצגת צריכה להציג טיעון אמיתי ומאוזן ולהוביל למסקנה שנשענת על הנתונים בלבד.

בסוף, הפק מסמך Markdown אחד עם מבנה המצגת: כותרת לכל שקף, נקודות התוכן של כל שקף, ואיזה נתון או מסמך מהתיקייה תומך בכל שקף. זה מסמך המסירה שאעביר לשלב ההפקה. אל תעצב מצגת בשלב הזה — רק את המבנה.

כתוב הכול בעברית. חצים פונים שמאלה. בלי אימוג'ים. משפטים מלאים, בלי לערבב עברית ואנגלית באותו משפט.
```

### שלב 2 — POWER מפיק את המצגת

אחרי ש-Point יחזיר לכם את המבנה המסודר של המצגת, הדביקו את הפרומפט הבא. תצטרכו לבחור יעד אחד, PowerPoint או Google Slides, וכן סגנון עיצוב אחד מתוך השלישייה:

```
קח את מבנה המצגת ב-Markdown שיצרת בשלב הקודם והפק ממנו מצגת מלאה.

יעד: [בחרו אחד — PowerPoint, או Google Slides].

סגנון עיצוב: [בחרו אחד מהשלישייה — מינימלי ונקי / נתונים וגרפים במרכז / מנהלים ומסר חד].

מותג TUESDAY: צהוב #FDB913, שחור #0B0B0B, לבן לטקסט על רקע שחור. בלי צבע שלישי. טיפוגרפיה כבדה, גאומטרית, אותיות גדולות. הסימן הוא רונה Tiwaz בצהוב, אקסנט קטן ולא מודפס על הכול.

הקפד שכל שקף יישען על הנתון או המסמך שצוין לו במבנה. אל תמציא מספרים חדשים — השתמש רק במה שהיה בתיקייה. בסוף, השקף האחרון מציג את ההכרעה ואת הנימוק שלה.

עברית. חצים שמאלה. בלי אימוג'ים. משפטים מלאים בלשון עניינית ופורמלית, בלי לפנות בגוף שני.
```

## בדיקה עצמית

איך תדעו שהצלחתם?
- אם Point סינן את הרעש לפני שהוא התחיל לעבוד, ואמר לכם בדיוק מה הוא זרק ולמה.
- אם Point עצר לראיין אתכם לפני שהוא בנה את המבנה, ולא קפץ ישר לייצר מצגת.
- אם כל שקף נשען אך ורק על נתון או מסמך שהיה בתיקייה, בלי מספרים מומצאים.
- אם המצגת מציגה את שני הצדדים של הוויכוח ומכריעה לפי הנתונים בלבד, ולא לפי דעה.

זכרו: אם נתקעתם בעיצוב, אתם צריכים לחזור ל-POWER. אם נתקעתם בתוכן, אתם חוזרים ל-Point. אל תילחמו ידנית בשקף בודד.

## הדבר השלישי — עמוד HTML

בכיתה הראינו שיש גם יעד שלישי לאותו תהליך בדיוק: עמוד HTML חי. לקחנו את התוכן של מצגת ה-onboarding לעובד חדש ב-TUESDAY, והפעם POWER הוציא לנו דף שזז, במקום מצגת רגילה. ראינו שם קונפטי, תנועה ותחושת ברוך הבא אמיתית. זו הייתה הפגישה הראשונה שלכם עם HTML.

את החלק הזה אתם לוקחים איתכם הביתה: קחו תוכן שכתבתם, הריצו אותו דרך Point ו-POWER כשהיעד הוא HTML ב-Claude הרגיל, ותראו איך נבנה לכם עמוד חי ונושם. זו בדיוק הדלת למה שנעמיק בו בהמשך הקורס.

## הרעיון, בשורה אחת

אנחנו לא מדברים כאן על שלושה כלים נפרדים, אלא על דיסציפלינה אחת ברורה: התוכן תמיד בא קודם. Point פותר לנו את הקושי האמיתי, POWER עושה את החלק הקל, ואותו תוכן יכול לצאת ב-PowerPoint, ב-Google Slides, או כעמוד HTML. התוכן הוא הניצחון שלנו. הבנייה היא רק הזנב.
