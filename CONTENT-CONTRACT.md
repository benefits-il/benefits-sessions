# CONTENT-CONTRACT — איך כותבים תוכן לאתר המסירה

הקובץ הזה מסביר לכל מי שכותב תוכן (בן או סוכן) איך לכתוב את קובצי ה-Markdown כדי שייצקו נכון לאתר. הכלל הפשוט: **כותבים MD רגיל, מוסיפים בלוק frontmatter קצן בראש, וזורקים את הקובץ לתיקייה הנכונה.** המנוע (`viewer.html`) עושה את כל השאר — כותרת ממותגת, סלוט סרטון, כפתורי העתקה, הורדות, וניווט.

## איפה הקובץ יושב

כל יחידה יושבת תחת `content/session-<N>/<unit-slug>/`, ובתוכה:

- `practice.md` — עמוד התרגול.
- `learning-chapter.md` — פרק הלמידה.
- `index.html` — עמוד היחידה (hub) שמקשר לשניהם. נבנה מהתבנית (ראו למטה).
- קבצים להורדה (תמלול, מאגר, PDF) — יושבים **באותה תיקייה** לצד ה-MD.

## ה-frontmatter (החוזה)

בראש כל קובץ תוכן, בין שתי שורות `---`:

```
---
type: practice
unit: chains
session: 2
phase: בבית
parent: content/session-2/chains/index.html
parentLabel: שרשור סוכנים
title: תרגול — שרשור סוכנים
studio: [Index]
video: TBD
downloads: [{"label":"תמלול הפגישה","href":"meeting-transcript.md"}]
links: [{"label":"סגננו שם לסוכן","href":"https://benefits-il.dev/agent-names/"}]
---
```

מה כל שדה עושה:

- **type** — `practice` או `learning-chapter`. קובע את ה-chrome: `practice` מקבל סלוט סרטון בראש; `learning-chapter` מקבל רוחב קריאה נוח.
- **title** — הכותרת הגדולה בראש העמוד (וגם כותרת הטאב בדפדפן).
- **session** — מספר המפגש. בונה את ה-breadcrumb ואת הקישור חזרה למפגש.
- **unit** — ה-slug באנגלית (chains, deep-research). שם-התיקייה.
- **parentLabel** — שם היחידה בעברית. מופיע ב-kicker וב-breadcrumb.
- **parent** — נתיב לעמוד היחידה (hub), **יחסית לשורש האתר**. הכפתור "חזרה" וה-breadcrumb מצביעים לשם.
- **phase** — `הכנה` / `בכיתה` / `בבית`. מופיע ב-kicker. לא חובה.
- **studio** — מערך של אותיות STUDIO (למשל `[Index]`, `[Scout]`). כל אחת הופכת ל-chip "STUDIO · …". **תמיד נוכח על התוכן.**
- **video** — קישור הטמעה לסרטון, או `TBD` כל עוד אין סרטון. ב-`TBD` מופיע סלוט ריק "הסרטון בקרוב". רלוונטי רק ל-`practice`.
- **downloads** — מערך JSON של `{label, href}`. הופך לשורת כפתורי הורדה מתחת לסרטון. ה-`href` יחסי לתיקיית הקובץ (הקובץ להורדה יושב לידו).
- **links** — מערך JSON של `{label, href}`. לינקים חיצוניים. `http(s)` נפתח בטאב חדש.
- **prev / next** — לא חובה. כל אחד `{"label":"…","href":"…"}`. בונה ניווט קודם/הבא בתחתית.

## כללי כתיבת הגוף

- **בלוק להעתקה = בלוק קוד עם שלושה גרשיים.** כל בלוק כזה מקבל אוטומטית כפתור "העתק" וצבעי תחביר. זה הדרך היחידה לתת ללומד טקסט להעתקה — לא טקסט בתוך פסקה.
- **טבלאות, רשימות, רשימות-משימה (`- [ ]`), ציטוטים, תמונות, וקישורים** — כל תחביר Markdown סטנדרטי (GFM) עובד ומעוצב במותג.
- **דיאגרמות Mermaid** — בלוק קוד עם השפה `mermaid` הופך לדיאגרמה מצוירת.
- **קישורים יחסיים** (לקובץ באותה תיקייה, למשל `meeting-transcript.md`) נפתרים אוטומטית מול תיקיית הקובץ. קובץ ההורדה חייב לשבת באותה תיקייה.
- עברית-first, חיצים שמאלה ←, בלי ערבוב עברית-אנגלית באמצע משפט, בלי אימוג'ים בגוף (אלא אם זה תוכן קיים בקול Story).

## להוסיף יחידה חדשה

1. צרו תיקייה `content/session-<N>/<unit-slug>/`.
2. כתבו בתוכה `practice.md` ו-`learning-chapter.md` עם ה-frontmatter למעלה.
3. העתיקו לתוכה את קבצי ההורדה (אם יש).
4. צרו `index.html` (hub) מהעתקת hub קיים — שנו את הכותרת, ה-chip של STUDIO, והקישורים ל-`viewer.html?doc=…` של היחידה. **כלל קבוע:** התרגול הוא הקישור הראשי למעלה (ההכנה נעשית דרך סדר התרגילים). פרק הלמידה תמיד יושב בתחתית, בתוך `<div class="chapter-aside">` (מתחת לבורדר מפריד), בשם **"פרק למידה מלא"**, עם הסבר שזה לקריאת כל הסיפור.
5. הוסיפו כרטיס `unit-card` לעמוד המפגש (`session-<N>.html`) ברצועה הנכונה (הכנה / בכיתה / בבית).

עמודי הקליפה (שער, אינדקס שיעורים, עמוד מפגש, hub) הם HTML סטטי שנבנה ביד מהתבניות — לא MD. רק התרגול ופרק הלמידה הם MD שעובר במנוע.
