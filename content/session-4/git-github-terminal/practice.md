# בבית — טרמינל, `Git` ו-`GitHub`

## מה עושים כאן

זו ההכנה הראשונה לשיעור. שלושה סרטונים קצרים, וכל אחד נגמר בתרגיל קטן שאתם עושים בעצמכם, על המחשב שלכם.

הסדר בנוי כמו מדרגות: קודם פותחים את ה-`Terminal` ומגלים שהוא לא מפחיד, אחר כך הופכים תיקייה למכונת-זמן ששומרת גרסאות, ובסוף מעלים את הכול לענן. בסוף הדרך תהיה לכם תיקייה אחת מגובה ב-`GitHub`, עם היסטוריה מלאה של כל שינוי. את זה אתם מביאים איתכם לכיתה.

אתם לא צריכים לדעת שום דבר מראש. כל פקודה שצריך מחכה כאן למטה, מוכנה להעתקה. פשוט עשו כל שלב יחד עם הסרטון.

> **נתקעתם? שאלו את `קלוד`.** אם משהו לא ברור, או שלא הבנתם מה שלב מסוים מבקש, פשוט שאלו את `קלוד`. צלמו מסך של מה שאתם רואים, הדביקו לו, ובקשו שיסביר: "מה אני רואה כאן ואיך אני ממשיך?". זה הכי מהיר, ו-`קלוד` יעבור איתכם צעד-צעד.

## 🎬 סרטון 1 · ה-`Terminal` הוא צ'אט, לא קוד

[[video:XjE89V8ZYFE]]

הסרטון מראה מה זה ה-`Terminal`, איך פותחים אותו, ואיך נעים בו בין תיקיות. צפו בו, ואז עשו את התרגיל.

### תרגיל 1 — פותחים את המסך השחור ובונים תיקייה

מה עושים:

1. פותחים את ה-`Terminal`. ב-`Mac`: מקש `Command` `+` רווח, מקלידים `Terminal`, ואז `Enter`. ב-`Windows`: מקש `Windows` `+` `X`, ובוחרים `PowerShell` (לא `CMD`).
2. שולחים שתי הודעות ראשונות — אחת ששואלת "איפה אני?", ואחת שבודקת ש-`Git` מותקן.
3. בונים תיקיית-עבודה ונכנסים אליה. כדי לראות שזה קורה באמת, פתחו את `Finder` (ב-`Mac`) או את `File Explorer` (ב-`Windows`) לצד ה-`Terminal` — התיקייה תופיע שם ברגע שתריצו את הפקודה.

מעתיקים ומריצים, שורה-שורה:

```
pwd
git --version
mkdir my-project
cd my-project
pwd
ls
```

מה השגתם: תיקייה בשם `my-project` שנוצרה מול העיניים שלכם בסייר הקבצים, `+` אישור ש-`Git` מותקן ומוכן.

בדיקה עצמית:
- אחרי `git --version` הופיע מספר גרסה (אם לא — התקינו `Git` מ-`git-scm.com`, וסגרו ופתחו את ה-`Terminal`).
- התיקייה `my-project` מופיעה בסייר הקבצים.
- אחרי `ls` לא הופיע כלום — וזה בדיוק נכון, התיקייה עדיין ריקה.

## 🎬 סרטון 2 · `Git` — מכונת הזמן

[[video:3pzqmluCML0]]

הסרטון מסביר מה זה `Git` ולמה הוא מחסל את בלגן הגרסאות, ואז עושה `commit` ראשון. צפו בו, ואז עשו את התרגיל.

### תרגיל 2 — הופכים את התיקייה למכונת-זמן

מה עושים:

1. הופכים את `my-project` ל-`repository`, ומגדירים פעם אחת מי אתם (השם והאימייל שלכם, אותו אימייל שאיתו תירשמו ל-`GitHub`).
2. יוצרים קובץ הערה קטן, כדי שיהיה מה לשמור.
3. לוקחים צילום-מצב ראשון (`commit`), ומסתכלים בהיסטוריה.

מעתיקים ומריצים. בשתי שורות ה-`config`, החליפו את השם והאימייל בשלכם:

```
git init
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
echo "ההערה הראשונה שלי" > notes.txt
git status
git add .
git commit -m "Initial commit"
git log
```

מה השגתם: `commit` ראשון ששמור לתמיד. ב-`git log` אתם רואים אותו — עם השם, התאריך וההודעה שכתבתם.

בדיקה עצמית:
- אחרי ה-`commit` הופיעה שורה עם `1 file changed` או `root-commit`.
- ב-`git log` מופיע הצילום הראשון שלכם.
- אם המסך נתקע עם `(END)` בתחתית — לחצו על `q` כדי לצאת.

הערה קטנה: השתמשו במרכאות רגילות, לא במרכאות "חכמות" שמועתקות ממסמך — הן שוברות את הפקודה.

## 🎬 סרטון 3 · `GitHub` — הענן

[[video:0GKtC2Uti0U]]

הסרטון מסביר מה זה `GitHub`, למה זה לא עוד `Google Drive`, ואיך דוחפים את העבודה לענן. צפו בו, ואז עשו את התרגיל.

### תרגיל 3 — פותחים חשבון `GitHub` ודוחפים לענן

מה עושים:

1. פותחים חשבון חינמי ב-`github.com`. שם-משתמש מקצועי וברור (הוא יופיע על כל העבודות שלכם), ואותו אימייל שהגדרתם ב-`Git`.
2. יוצרים `repository` ריק חדש — לחצו על הפלוס ואז `New repository`, תנו לו שם, והשאירו אותו **בלי `README`** כדי שלא ייווצרו התנגשויות. העתיקו את כתובת ה-`HTTPS` שלו (לא `SSH`).
3. מחברים את התיקייה שלכם לענן ודוחפים. החליפו את `<URL>` בכתובת שהעתקתם:

```
git remote add origin <URL>
git branch -M main
git push -u origin main
```

ייפתח חלון — לחצו על `Sign in with your browser`, אשרו בדפדפן, וזהו.

4. עכשיו הלולאה היומיומית: פתחו את `notes.txt`, שנו בו משהו, שמרו, ודחפו עדכון:

```
git add .
git commit -m "Update notes"
git push
```

מה השגתם: התיקייה `my-project` יושבת ב-`github.com` עם שתי גרסאות בהיסטוריה. לחצו על `Commits` בעמוד ה-`repository` — אתם רואים את `Initial commit` ואת `Update notes`, ויכולים לפתוח כל אחת ולראות בדיוק מה השתנה.

בדיקה עצמית:
- ה-`repository` מופיע ב-`github.com` עם הקבצים שלכם בפנים.
- יש שני `commits` בהיסטוריה.
- הפעם השנייה הספיק `git push` לבד, בלי הגדרות — כי כבר חיברתם את ה-`origin`.

הערה קטנה: אם הדחיפה נדחתה (`non-fast-forward`), כנראה סימנתם `README` ביצירת ה-`repository`. צרו אותו מחדש בלי `README`, ונסו שוב.

## מה הלאה

זהו, אתם מוכנים לכיתה. יש לכם `Terminal` שאתם לא מפחדים ממנו, מכונת-זמן מקומית, וחשבון `GitHub` עם עבודה מגובה בענן.

ואם משהו נתקע בדרך — קודם כול שאלו את `קלוד` (צילום-מסך `+` "מה אני רואה ואיך ממשיכים?"). ואם עדיין תקוע, שלחו צילום מסך בקבוצה, ונפתור את זה יחד עוד לפני השיעור. עדיף להגיע כשהכול כבר עובד.
