# HOW-TO — איך מעלים ומעדכנים תוכן באתר המסירה

מדריך מדויק לבן. כל שינוי שעולה ל-git מתעדכן אוטומטית ב-**benefits-il.dev/benefits-sessions** תוך כדקה. עבוד תמיד בעותק המקומי בתיקייה הזו, בדוק מקומית, ואז דחוף.

---

## איך האתר עובד (חצי דקה)

שתי שכבות:

1. **תוכן** — קובצי `.md` תחת `content/session-N/<unit>/`. המנוע `viewer.html` מרנדר אותם לעמוד מעוצב. **אתה כותב MD, המנוע עושה את העיצוב.**
2. **קליפת ניווט** — עמודי HTML סטטיים: `index.html` (שער), `sessions.html` (אינדקס), `session-N.html` (עמוד מפגש), ו-`content/.../index.html` (עמוד יחידה / hub).

רוב העדכונים = עריכת קובץ MD אחד. פירוט ה-frontmatter המלא ב-[CONTENT-CONTRACT.md](CONTENT-CONTRACT.md).

---

## תרחיש 1 — לעדכן תוכן קיים (הכי נפוץ)

1. פתח את קובץ ה-MD תחת `content/session-N/<unit>/` (`practice.md` או `learning-chapter.md`).
2. ערוך את הטקסט. **בלוק להעתקה = שלושה גרשיים** (` ``` `). הוא מקבל לבד כפתור "העתק".
3. בדוק מקומית (למטה), ודחוף (למטה).

לא צריך לגעת ב-HTML. הכותרת, ה-chip, הסרטון, ההורדות והניווט נבנים אוטומטית מה-frontmatter.

---

## תרחיש 2 — להוסיף או להחליף סרטון של יחידה

ב-frontmatter של `practice.md`, שנה את שורת `video:`:

- אין סרטון עדיין → `video: TBD` (מציג סלוט "בקרוב").
- יש סרטון → `video: <מזהה-יוטיוב>` — 11 התווים מה-URL (אחרי `v=` או אחרי `youtu.be/`).

המנוע בונה לבד הטמעת youtube-nocookie עם כתוביות עברית. דוגמה:

```
video: baSn1qXMeYw
```

דחוף, וזהו.

---

## תרחיש 3 — להוסיף יחידה חדשה למפגש קיים

1. צור תיקייה `content/session-N/<unit-slug>/` (ה-slug באנגלית, למשל `chains`).
2. צור בתוכה `practice.md` ו-`learning-chapter.md` עם frontmatter מלא — **הכי קל להעתיק יחידה קיימת ולשנות**. שדות חובה: `type`, `title`, `session`, `unit`, `phase`, `parent`, `parentLabel`, `studio`; לתרגול גם `video`, `downloads`, `links`.
3. העתק לתיקייה את קבצי ההורדה (תמלול / מאגר / PDF) שה-`downloads` מצביע אליהם.
4. צור `index.html` (hub) בהעתקת hub קיים. עדכן כותרת, chip STUDIO, ושני הקישורים. **שמור על הכלל הקבוע:** התרגול הוא הקישור הראשי למעלה; פרק הלמידה יושב בתחתית בתוך `<div class="chapter-aside">` בשם "פרק למידה מלא".
5. הוסף כרטיס `unit-card` לעמוד המפגש (`session-N.html`) ברצועה הנכונה (הכנה / בכיתה / בבית).
6. בדוק מקומית, דחוף.

---

## תרחיש 4 — להוסיף מפגש חדש

1. צור `session-N.html` בהעתקת `session-2.html` (שלוש רצועות), עדכן כותרת ותוכן.
2. ב-`sessions.html`, החלף את כרטיס ה-stub של אותו מפגש בקישור פעיל: הסר `is-locked`, הפוך את ה-`<div class="session-card">` ל-`<a class="session-card" href="session-N.html">`, ושנה את שורת התחתית ל"כניסה למפגש".
3. הוסף את היחידות שלו תחת `content/session-N/` (תרחיש 3).

---

## לבדוק מקומית לפני שדוחפים (חובה)

הדפדפן חוסם טעינת MD מקומי ב-`file://`, אז צריך שרת. מתוך התיקייה הזו, הרץ:

```
python -m http.server 8000
```

פתח **http://localhost:8000** ועבור על מה ששינית. (אם אין python: `npx serve`.)

---

## לדחוף live (git)

מתוך התיקייה הזו:

```
git add -A
git commit -m "תיאור קצר של השינוי"
git push
```

תוך כדקה זה live ב-**benefits-il.dev/benefits-sessions**.

אם אתה עובד עם Claude Code בתיקייה הזו — פשוט תגיד "תדחוף את זה" והסוכן יעשה לפי [CLAUDE.md](CLAUDE.md), כולל הבדיקה המקומית.

---

## כללי זהב (אל תשבור)

- **קובץ הורדה יושב תמיד באותה תיקייה כמו ה-MD שמקשר אליו.** קישורים יחסיים בגוף ה-MD נפתרים מול תיקיית הקובץ.
- **נתיבי `?doc=` הם תמיד מהשורש:** `content/session-N/<unit>/<file>.md`.
- **אל תיגע ב-`viewer.html` וב-`delivery.css`** אלא אם אתה משנה את המנוע או את העיצוב — הם משותפים לכל העמודים, וריפו המנחה התאום אמור לרשת אותם נקי.
- עברית, חיצים שמאלה ←, בלי אימוג'ים בכותרות, בלי ערבוב עברית-אנגלית באמצע משפט.
- בדוק מקומית לפני כל push.
