# הכנה — מה זה Plugin

## סרטון היחידה

<!-- video set: DMX81pA1dYE (youtu.be/DMX81pA1dYE, unlisted) — המנוע בונה את ה-embed מ-video: ב-frontmatter בשלב ההעברה לאתר -->

הסרטון מסביר את המושג בקצרה. אפשר לצפות בו קודם, או פשוט להתחיל מהתרגול שלמטה.

## לפני שמתחילים

עד עכשיו בניתם סקילים, חיברתם connectors, הרצתם פקודות והקמתם סוכנים.
כל אחד מהם עמד בפני עצמו, וכל אחד דרש התקנה נפרדת.
plugin הוא פשוט אריזה. דרך לאסוף כמה מהכלים האלה יחד לחבילה אחת, ולהתקין אותם במכה אחת.

plugin הוא לא מושג חדש ללמוד. הוא אריזה של דברים שאתם כבר מכירים.
אם תפתחו plugin, תמצאו בתוכו את הרכיבים המוכרים האלה:

- **סקיל** — יכולת אחת. דבר שהמערכת יודעת לעשות.
- **חיבור** — דלת לכלי חיצוני (connector).
- **פקודה** — פעולה שמתחילה בלוכסן ומפעילה משהו בלחיצה.
- **סוכן** — פרסונה עם תפקיד (sub-agent) שמטפלת במשימה מקצה לקצה.

מה צריך להכין: Claude Desktop עם Cowork, בדיוק כמו בסוף היחידה הקודמת.
ההתקנה של plugin נעשית בתוך Cowork. אם הוא פתוח אצלכם, אפשר להתחיל.

(רק הערה חשובה לפני שמתחילים: כאן רק מתקינים ומשחקים עם plugin מוכן. את הבנייה של plugin משלכם נעשה יחד בכיתה.)

כל החומרים שתצטרכו, ההורדות והפרומפטים המוכנים, יושבים בעמוד אחד: [עמוד החומרים של Tuesday](https://benefits-il.dev/tuesday). פותחים אותו בצד, ומתחילים.

## שני מסלולים

כדי שנעבוד על חברה אמיתית, נכיר את Tuesday. זו אפליקציית ניהול-עבודה לעצמאי שמריץ לבד את כל העסק. כל החומרים שתורידו הם של Tuesday.

התרגול בנוי משתי תחנות מחוברות, ויש שני מסלולים:

- **מסלול א' — תחנה 1 בלבד.** זה מה שכולם עושים. מתקינים plugin אחד, יוצרים את קול-המותג של Tuesday, ופוגשים שניים מהרכיבים. מספיק כדי להבין מה זה plugin.
- **מסלול ב' — תחנה 1 plus תחנה 2.** המסלול המלא. ממשיכים מהקול שיצרתם, מתקינים plugin שני, ומפיקים תוכן אמיתי. כאן פוגשים את כל ארבעת הרכיבים, ורואים איך התוצר של plugin אחד הופך לחומר-הגלם של הבא.

התוצר של התחנה הראשונה הוא חומר הגלם של השנייה. לכן כולם מתחילים מתחנה 1.

## תחנה 1 — יוצרים את קול-המותג

הרעיון: מתקינים plugin מוכן מהספרייה של קלוד, שנקרא **brand-voice**.
נותנים לו את כל החומרים המפוזרים של Tuesday.
הוא מזקק מהם קול עקבי אחד, ומפיק מסמך הנחיות-קול.

צעד אחר צעד:

1. מתקינים את ה-plugin **brand-voice** מהספרייה הרשמית של קלוד, **anthropics/knowledge-work-plugins**. יש שתי דרכים, בוחרים אחת. **דרך הממשק:** ב-Cowork נכנסים ל-Customize, ללשונית Plugins, ובאזור Personal plugins לוחצים על הפלוס ובוחרים Add marketplace, מקלידים את שם-הספרייה anthropics/knowledge-work-plugins, לוחצים Browse plugins, ומתקינים מתוכה את brand-voice. **או דרך שורת-הפקודה** — פותחים את /plugin ומריצים את שתי השורות:

```
/plugin marketplace add anthropics/knowledge-work-plugins
/plugin install brand-voice@knowledge-work-plugins
```

2. ב[עמוד החומרים של Tuesday](https://benefits-il.dev/tuesday) מורידים את **חבילת תחנה 1** ופותחים את ה-zip. בתוכה תיקייה בשם **raw**: החומרים המפוזרים של Tuesday — מיילים, פוסטים והכרזות. חלקם טובים, חלקם משעממים ותאגידיים. זה הבלגן שממנו ה-plugin יזקק קול.
3. מצרפים לצ'אט את הקבצים מתוך התיקייה raw, ומריצים את הפקודה **/brand-voice:discover-brand**. את הפרומפט המדויק (שמכריח אותו לעבוד רק עם הקבצים שצירפתם) מעתיקים מ[עמוד החומרים](https://benefits-il.dev/tuesday).
4. אחרי שהפקודה מסיימת, מריצים **/brand-voice:generate-guidelines** ומקבלים מסמך הנחיות-קול. **שומרים אותו בשם `brand-voice.md`**. ניקח את המסמך הזה איתנו לתחנה 2.

מה ראיתם כאן: הופעלה חבילה אחת מהספרייה שיש בה **סקיל** ו**פקודות**.
היא לקחה ערימה של קבצים מבולגנים והפכה אותם לקול ברור.
אלה שניים מארבעת הרכיבים המוכרים, כבר בתחנה הראשונה.

אם עשיתם רק את תחנה 1, סיימתם את מסלול א'. מי שרוצה את המלא — ממשיך לתחנה 2.

## תחנה 2 — משתמשים בקול

הרעיון: מתקינים plugin רחב יותר ובגנרי בשם **brand-studio**.
נותנים לו את `brand-voice.md` שיצרתם, ועוד קבצים מערכת-המותג.
הוא מייצר מהם תוכן אמיתי: מייל, פוסט, קרוסלה או סרטון.
כאן גם פוגשים את הרכיב הרביעי: ה**חיבור**.

צעד אחר צעד:

1. מתקינים את ה-plugin **brand-studio** מהספרייה **benefits-il/brand-studio**. יש שתי דרכים, בוחרים אחת. **דרך הממשק:** ב-Cowork נכנסים ל-Customize, ללשונית Plugins, ובאזור Personal plugins לוחצים על הפלוס ובוחרים Add marketplace, מקלידים benefits-il/brand-studio, לוחצים Browse plugins, ומתקינים את brand-studio. **או דרך שורת-הפקודה** — פותחים את /plugin ומריצים את שתי השורות:

```
/plugin marketplace add benefits-il/brand-studio
/plugin install brand-studio@brand-studio
```

2. מוסיפים את חיבור **Higgsfield** (הוא מצייר ומפיק וידאו). ברשימת ה-Connectors בוחרים Add custom connector וקוראים לו Higgsfield. מדביקים את הכתובת `https://mcp.higgsfield.ai/mcp`, לוחצים Connect ומתחברים עם חשבון Higgsfield (חינם, בלי כרטיס אשראי). למיילים, ודאו ש-Gmail מחובר.
3. ב[עמוד החומרים של Tuesday](https://benefits-il.dev/tuesday) מורידים את **חבילת תחנה 2** ופותחים אותה. מצרפים לצ'אט שלושה קבצים: את `brand-voice.md` (הקול שיצרתם בתחנה 1), את `brand-sheet.md` (הצבעים, הסמל והעובדות), ואת הבריף של המלאכה שבחרתם. מעתיקים מאותו עמוד את הפרומפט המתאים ומדביקים.
4. נסו להריץ מלאכה אחת.
למשל `/welcome` — ייצור מייל ברוכים-הבאים ממותג, ויישמר כטיוטה ב-Gmail.
או `/post` — ייצור תמונה וכיתוב דרך Higgsfield.
הקול שיצרתם בתחנה 1 הוא זה שגורם לתוכן להישמע כמו Tuesday.

אלה חמש המלאכות שאפשר להריץ. לכל אחת יש בריף משלה בחבילה ופרומפט מוכן בעמוד:

- `/outreach` — מייל פנייה (הבריף: outreach-context.md)
- `/welcome` — מייל ברוכים-הבאים (welcome-points.md)
- `/post` — פוסט: תמונה וכיתוב (social-seed.md)
- `/carousel` — קרוסלה מרובת שקופיות (carousel-outline.md)
- `/reel` — סרטון דמות קצר (character-brief.md)

(בתוך חבילת תחנה 2 יש גם תיקיית backup עם `brand-voice.md` מוכן. זה גיבוי בלבד, למי שנתקע בתחנה 1. הדרך הנכונה היא לייצר את הקול בעצמכם.)

מה ראיתם כאן: עבודה עם ארבעת הרכיבים יחד.
יש שם סקיל, פקודה, סוכן, ו**חיבור** שפותח דלת לעולם שבחוץ.
ו-brand-studio הוא פשוט plugin אחד שאורז את כולם יחד.

## צ'קליסט לפני הכיתה

עברו על הרשימה. ברגע שתחנה 1 מסומנת, אתם מוכנים:

- התקנתם את brand-voice מהספרייה המובנית והפקתם את הנחיות-הקול של Tuesday (תחנה 1).
- שמרתם את ההנחיות בשם `brand-voice.md`.
- (מסלול ב', מי שהמשיך) התקנתם את brand-studio, חיברתם את Higgsfield, והרצתם לפחות מלאכה אחת שהשתמשה בקול שיצרתם (תחנה 2).

זה הכול.
עכשיו אתם יודעים ש-plugin הוא בסך הכול אריזה של רכיבים מוכרים.
ומי שהמשיך לתחנה 2 גם ראה איך התוצר של plugin אחד הופך לחומר הגלם של הבא בתור.
בכיתה נתחיל להרכיב plugin משלנו.
