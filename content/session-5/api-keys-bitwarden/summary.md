# סיכום · מפתח, כסף וכספת

## מה שלמדנו

1. **מפתח API הוא כרטיס אשראי דיגיטלי, לא סיסמה.** הוא אישור למוכ"ז (Bearer): המערכת לא בודקת מי מחזיק בו — מי שמחזיק, פועל בשמכם ומחייב אתכם. לכן מתייחסים אליו כמו לכרטיס אשראי: לא משאירים על הספסל.
2. **קודם כספת, אחר-כך מפתח.** ‏Bitwarden עם הצפנת אפס-ידע: הכול מוצפן על המחשב שלכם לפני שהוא עולה לענן, ורק סיסמת המאסטר פותחת. התאמנו על רשומת דמו — פריט Note עם שם ברור, תיקייה, והמפתח בשדה מוסתר (Hidden) בשם key — ומחקנו אותה. עכשיו הכספת מוכנה לאמיתי.
3. **את המפתח האמיתי משיגים ב-Google AI Studio.** ‏Get API key ← ‏Create API key ← המפתח מוצג פעם אחת ← ישר לרשומה בכספת, בלי תחנת ביניים ← **Restrict to Gemini API** — נעילה לשירות אחד, כך שגם מפתח שדלף לא שווה כלום במקום אחר.
4. **הכסף הטעון הוא רשת-הביטחון.** הקורס עובד על Gemini בתשלום — טעינה חד-פעמית של כ-15 דולר מכסה את כל תרגילי המדיה בשפע. הקרדיט נקנה מראש, בלי Auto-reload — אז מה שטענתם הוא תקרת-ההוצאה, עקרון ה-Hard Limit מובנה. בונוס: בשכבה בתשלום התוכן שלכם לא משמש לאימון המודלים של Google.
5. **משתמשים בלי לחשוף.** לא בקוד או ב-GitHub, לא בצ'אט או במייל, רק בכלי שבוטחים בו — והשאלה הנכונה היא תמיד "איך הכלי שומר את המפתח?". בדקנו את זה בפועל: שלפנו את המפתח מהכספת והפעלנו איתו אתר שניתח לנו PDF. שיתוף רק דרך Bitwarden Send.
6. **מפתח שדלף מבטלים, לא מצילים.** ‏Revoke בלוח הבקרה חוסם את הגישה תוך שניות ← יוצרים חדש ← מעדכנים את הכלים. ביטול הוא זול ומהיר — ולכן אפשר לעבוד בביטחון.

## השורה ששווה לזכור

> מפתח API הוא כרטיס אשראי דיגיטלי: בניתי כספת, יצרתי מפתח ששמור בה מהרגע הראשון, הגבלתי אותו, טענתי סכום שאני שולט בו — והפעלתי איתו כלי אמיתי. ואם הוא דולף, אני מבטל אותו בשניות.
