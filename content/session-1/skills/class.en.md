# In Class — Skills

## Build a skill live, then look inside one

We build a skill from scratch together, save it, run it again in a fresh chat — and then open an existing skill to see what is really inside it.

## The instructions

1. Pick three random words and ask Claude for a short story. Type this:

   > Pick three random words in English, unrelated to one another, separated by commas.
   >
   > Write me a short five-line story from these words.

2. Turn that into a skill. Type:

   > I want you to turn this into a Skill.

3. Answer Claude's questions as it builds the skill — the language, the tone, and the trigger that should set it off.
4. Save the skill with "save skill". If it says one already exists, choose "update and replace".
5. Open a new chat, run the skill with a slash, give it three words, and watch it work again.

## Discovery (optional, if you finish early)

Open one existing skill — any skill except Skill creator — and find the two things inside it:

- It is a folder that holds more than the instruction file (for example a sub-folder of themes or fonts).
- The instruction file itself is written in MD, a plain-text format with marks like hash signs and asterisks.

## Desired result

One skill that you built yourself, saved in Customize, and triggered again in a fresh chat — plus, if you reached the discovery, having seen with your own eyes that a skill is a folder and that its instruction file is MD.
