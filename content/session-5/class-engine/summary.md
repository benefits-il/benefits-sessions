# סיכום · בונים מנוע

## מה שלמדנו

1. **MCP זו הדרך לשחרר דברים החוצה.** מייל ממותג מתוך קלוד, סרטון שעולה ליוטיוב, הקלטות שנשלפות מזום, יומן שמתעדכן — כל אחד מהחיבורים האלה הוא MCP אחד. היום בנינו את הראשון: מנוע-מדיה שעוטף את Gemini ונותן לקלוד שש יכולות — תמלול, ניתוח, תמונות, וידאו, דיבור ומוזיקה. בלי שורת קוד משלנו.
2. **לכל MCP יש אותו קו-ייצור.** קלוד מתחבר לשירות, עוטף אותו בשרת MCP, מגדיר את הכלים, שם את המפתח בקובץ env — ובודק שהכול חי. ראינו אותו פעם אחת, ומעכשיו נזהה אותו בכל חיבור שנבנה.
3. **הסקיל הוא ארט-דיירקטור אישי.** בנאי בונה אותו מחומרי המותג שכבר יש לנו בתיקייה, והוא מתאים את עצמו לבקשה: ביקשנו פרומפט — הוא מנסח פרומפט מלא ומדויק; ביקשנו תמונה או מדיה — הוא מפעיל ישירות את כלי המנוע.
4. **נקודת המפגש היא תמונה אחת על-מותג.** מבקשים מהסקיל תמונת פתיחה, והוא מפעיל את generate_image של המנוע ומחזיר תמונה עם הצבעים שלנו. זו קריאה בתשלום, כשבעה סנט (כ-$0.067). כשהתמונה מופיעה — יש לנו מנוע, ויש לנו סקיל.

## השורה ששווה לזכור

> MCP זו הדרך שלכם לשחרר החוצה דברים.

## תזכורת קטנה

את המפתח מדביקים רק בקובץ ה-env שקלוד פותח לכם, אף פעם לא בצ'אט — מה שמודבק בצ'אט נשלח ונשמר, מה שמודבק בקובץ נשאר במחשב שלכם. ה-15$ שטענתם בהכנה מכסים את כל תרגילי הקורס בשפע. וזכרו: ייצור תמונה מחזיר צבעים קרובים, לא מדויקים — לנכסים שחייבים צבע או טקסט מדויק, כמו לוגו, כרטיסים וטיפוגרפיה, הכלי הנכון הוא קוד, לא ייצור.
