# At home — Building a Plugin

## First, finish what you started

In class you started building a plugin, either in Linear or in Canva. The first thing to do at home is finish it. Open Plugin Create from wherever you stopped, answer the questions that were still open, install the plugin in Cowork, and run it once on something real. Not a demo, something of your own: your stack of tasks onto the board, or the facts about your product onto a sales page. The moment you see the result come out, you've finished the first part.

## And now, build more — this time for your world

In class we stayed in the world of Tuesday. At home we move to your world. Below you'll find a stash of twenty projects, split into four groups. Each one is a complete plugin with a ready opening prompt: copy it into Plugin Create, answer the questions, and you get a tool.

Don't build them all. Build one or two that will genuinely help you, from your real world, work or home. After the stash there's a table to help you choose smartly, so that a few builds together touch every kind of capability.

## The stash — twenty projects

### Group A · Fun and personal

**1. The perfect trip planner** — takes a destination, dates, and a budget, and builds a day-by-day itinerary with flights, hotels, and recommendations.

```
I want to build a plugin that plans trips. It takes a destination, dates, and a budget from me, searches for flights and hotels through Expedia and Booking, cross-checks recommendations from Tripadvisor, and produces a document with a day-by-day itinerary and a cost estimate. Help me build it step by step.
```

**2. The music curator / personal DJ** — describe a mood or an occasion, and it builds a playlist on Spotify.

```
I want to build a plugin that's a personal DJ. I describe a mood or an occasion to it, and it builds me a playlist on Spotify, explains why it picked each track, and can also clean up my library. Help me build it.
```

**3. The gigs and events finder** — finds shows that match your taste, checks tickets, and syncs to your calendar.

```
I want a plugin that finds me gigs and events that match my taste and my area, checks tickets on StubHub, and adds the ones I'm interested in to my Google Calendar. Help me build it.
```

**4. The event planner** — plans a party end to end: guest list, schedule, shopping, and invitations.

```
I want a plugin that plans events. I tell it which event and for how many people, and it builds a checklist, a schedule, a shopping list, and draft invitations by email. Help me build it.
```

**5. The home chef** — tell it what's in the fridge and how many people, and it builds a menu and a shopping list.

```
I want a plugin that's a home chef. I tell it what I have in the fridge and how many people, and it suggests a menu and builds a tidy shopping list. Help me build it.
```

What sets them apart is the capabilities they touch beyond that: a connection to an external service, automatic scheduled runs, working on a folder of files, searching the web, and an agent that carries out a task end to end. ## The simple rule behind the table

Two projects are the complements:

- **17, the morning brief** is the only one that runs on an automatic schedule, and it also brings a connection to the calendar and to email. Without it, you'll never touch an automatic run.
- **13, the content machine** brings working on files, searching the web, and an agent that carries out end to end, three capabilities in one shot.

So the rule is simple: first build the project that genuinely helps you, then add 17 and 13. Three builds, and you've touched every kind of capability, connection, scheduling, files, web, end-to-end agent, and a professional deliverable. And if you've already built 12 or 13, you're almost there, all you're missing is the scheduling from 17.

This isn't an exercise for the sake of an exercise. Every build like this is a real tool that stays with you after the lesson.
