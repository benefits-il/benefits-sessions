# בבית — MCP, החיבור האוניברסלי

## מה עושים כאן

זו ההכנה השנייה לשיעור 5. בהכנה הקודמת בניתם כספת, יצרתם מפתח וטענתם קרדיט — ובכיתה תבנו עם זה מנוע שלם משלכם. אבל כדי לבנות מנוע, צריך קודם להבין את השפה שבה כלים מתחברים לקלוד. השפה הזאת נקראת MCP.

כאן עושים שני דברים: מבינים מה זה MCP ולמה הוא משנה את הכללים — ואז מחברים MCP אמיתי ראשון, בקליק, בחינם.

> **נתקעתם? שאלו את קלוד.** אם משהו לא ברור, או שמסך אצלכם נראה אחרת מבהוראות, צלמו מסך, הדביקו לקלוד, ובקשו שיסביר: "מה אני רואה כאן ואיך אני ממשיך?". זה הכי מהיר, וקלוד יעבור איתכם צעד-צעד.

### מה שאתם צריכים קודם

- Claude Code על המחשב — שם מתבצע החיבור. התקנתם אותו כבר בהכנת שיעור 4.
- חשבון Google פעיל — ההתחברות היא עם login של גוגל, בלי מפתח.
- מסמך ההתקנה של NotebookLM MCP — מחכה לכם בעמוד הזה. נותנים אותו לקלוד, והוא מתקין.

## הבעיה: כאוס האינטגרציה

עד היום, כל פעם שרצו לחבר כלי AI למקור מידע או לכלי חיצוני, היה צריך לכתוב קוד ייעודי בדיוק לחיבור הזה. מודל אחד כפול כלי אחד — חיבור אחד לתחזק. עכשיו תכפילו: חמישה מודלים כפול עשרה כלים — חמישים חיבורים. לזה קוראים בעיית N על M, והיא בלתי אפשרית לתחזוקה.

והפתרון פשוט ויפה. תחשבו על USB: פעם לכל מכשיר היה כבל אחר, היום יש שקע אחד אוניברסלי שמחברים אליו הכול. **MCP — ראשי תיבות של Model Context Protocol — הוא בדיוק זה: ה-USB של עולם ה-AI.** שפה אחת שכל מודל וכל כלי מדברים. מחברים מקור מידע או כלי פעם אחת, וכל בינה מלאכותית שתומכת בתקן יכולה להשתמש בו מיד.

ומי משתתף בחיבור? שלושה תפקידים, ותפגשו אותם שוב בכיתה:
- **Host** — האפליקציה שאתם עובדים בה. אצלנו: קלוד.
- **Client** — המתורגמן שיושב בתוכה ומנהל את הפרוטוקול.
- **Server** — ה"תקע" שמתחבר לכלי או למידע ומשדר אותם פנימה.

## רגע, למה לא API רגיל? ולמה לא קונקטור?

שתי שאלות שנשמעות טכניות, אבל התשובות עליהן הן כל הסיפור.

**למה לא פשוט API?** API רגיל הוא כמו מכונת חטיפים בלי חלון זכוכית: גם לא רואים מה יש בפנים, וגם אין דרך לשאול את המכונה — חייבים לדעת מראש בדיוק איזה כפתור ללחוץ. ככה API — המתכנת מקודד מראש כל פעולה, והמודל עיוור למה שהשירות מציע. **שרת MCP, לעומת זאת, הוא כמו הבחור הנחמד בקיוסק השכונתי:** הוא יודע בדיוק אילו פחיות, קרטיבים וגלידות יש אצלו — ויכול אפילו להכין לכם טוסט עם התוספות שאתם אוהבים. שרת MCP מתאר את עצמו לקלוד — קלוד מגלה לבד אילו כלים קיימים, מבין מה כל אחד עושה, ומבקש בדיוק את מה שהוא צריך.

**ולמה לא קונקטור מוכן?** את הקונקטורים כבר פגשתם בקורס — והם נוחים, אבל נעולים: הספק בנה אותם לפלטפורמה שלו, ומה שאין ברשימה — אין. ‏MCP הוא שלכם: מחברים או בונים פעם אחת, וזה עובד עם כל כלי עתידי שתומך בתקן. ניידות ובעלות מול נעילת ספק.

**אז מתי בונים MCP משלכם?** ארבעה סימנים: הלוגיקה ייחודית לכם ולא קיימת בכלי מדף · אתם רוצים שאותו כלי יעבוד מול קלוד, מול אוטומציות ומול סוכנים בלי נעילה · התהליך מורכב ודורש שרשרת פעולות · השליטה והאבטחה קריטיות. ואם אף אחד מהסימנים לא מתקיים — קונקטור מוכן זה בדיוק מה שאתם צריכים. לא בונים כשלא צריך.

טבלת ההשוואה המלאה של שלוש הדרכים מחכה לכם בדף הסיכום של היחידה.

## תרגיל — מחברים MCP ראשון: NotebookLM

למה דווקא NotebookLM? כי הוא חינם לגמרי — מתחברים עם חשבון גוגל, בלי מפתח API ובלי כרטיס אשראי — וכבר עבדתם איתו בקורס. זו הדרך הכי נקייה לראות MCP חי.

מה עושים:

1. פותחים את Claude Code בתיקייה כלשהי במחשב.
2. נותנים לקלוד את [מסמך ההתקנה של NotebookLM MCP](nbl-mcp-install.md) ומבקשים ממנו: "תתקין לי לפי המסמך הזה".
3. קלוד מתקין את החיבור ומלווה אתכם צעד-צעד. אם הוא מבקש לפתוח מחדש את Claude Code — עושים את זה.
4. כשמתבקשים — מתחברים עם חשבון ה-Google שלכם בחלון שנפתח, ומאשרים הרשאות. זה כל האימות: בלי מפתח, בלי טופס.
5. ובודקים שזה חי — מבקשים מקלוד משהו שדורש את NotebookLM, למשל:

```text
רשום לי אילו מחברות יש לי ב-NotebookLM
```

או, אותו פרומפט באנגלית:

```text
List the notebooks I have in NotebookLM
```

תסתכלו מה קורה: קלוד מזהה לבד שיש לו כלי חדש, פונה אליו, ומחזיר תשובה אמיתית מהחשבון שלכם. לא כתבתם קוד, לא הגדרתם כלום — חיברתם פעם אחת, וקלוד כבר יודע להשתמש. זה ה-USB בפעולה. (אין לכם עדיין מחברות? בקשו מקלוד ליצור אחת — גם זה עובד.)

מה השגתם: ‏MCP ראשון מחובר ועובד, וראיתם במו עיניכם את הרגע שבו קלוד מגלה כלי חדש ומשתמש בו לבד.

בדיקה עצמית:
- החיבור של NotebookLM מופיע בקלוד.
- ביקשתם פעולה, וקלוד הפעיל את הכלי והחזיר תשובה אמיתית מהחשבון שלכם.

## מה הלאה

חיברתם MCP מוכן שמישהו אחר בנה. בכיתה נעשה את הקפיצה הגדולה: תבנו MCP משלכם — מנוע מדיה סביב Gemini, עם המפתח שכבר מחכה לכם בכספת — ותבינו למה זה משנה את מה שאפשר לבנות. בהכנה הבאה: טעימה קצרה מעולם שלם של חיבורים מוכנים.

ואם משהו נתקע — קודם כול שאלו את קלוד, ואם עדיין תקוע, שלחו צילום מסך בקבוצה, ונפתור יחד עוד לפני השיעור.
