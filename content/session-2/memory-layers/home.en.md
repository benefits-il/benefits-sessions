# At home — Memory Layers

In class we built Benefit and gave it two skills, save and retrieve. It already works. Now, at home, we complete the picture: the rest of its skills, everything we've learned so far, and a few real runs that turn it from yours-on-paper into yours-for-real. Four tasks, in this order.

## Task one: finish Benefit, and give it the knowledge

If you didn't finish building Benefit in class, start here. Open a new project in Claude, paste the ready-made agent into the project instructions, and connect Notion with read and write permission. Then upload all the knowledge files, including the course-structure file. It's all on one page: **benefits-il.dev/benefit-agent**.

And if you did build it in class, it's still worth stopping to check that everything is in. Paste this prompt to Benefit:

```
Who are you, and which knowledge files do you have right now? Go through them one by one, and tell me specifically whether you have the course-structure file and the unit materials. If anything is missing, tell me exactly what.
```

**Why this:** an agent without its knowledge files can't file anything in the right place, and the course-structure file is exactly what makes the filing automatic. Without it Benefit guesses; with it, it knows.

## Task two: the two remaining skills, and one run each

In class you installed save and retrieve. Two are left, and both are on the same download page. Install each one in Claude through Settings → Skills → Upload, and run it once on real material of your own.

**The first skill, consult and break down by STUDIO.** It takes a real task of yours and breaks it into the six stations, with a recommendation of which tool to use at each stage. Run it once:

```
I have a task: [your task in one line]. Break it down for me by STUDIO, and tell me which tool to use at each stage.
```

**The second skill, build a prompt by MICRO.** It builds you a clean, ordered prompt and hands it back ready to copy. Run it once:

```
I need a prompt that [what you want the prompt to do]. Build it for me by MICRO.
```

**Why this:** each skill is one more thing Benefit does for you. And you trust a skill only after you've seen it work on your own material, not before. One run is all it takes.

## Task three: every learning chapter, into Notion

So far Benefit knows what we saved together in class. Now we give it everything we've learned to date, so it has the full picture. Eight learning chapters, from both sessions:

From Session 1, foundations, skills, agents. And from the Session 2 prep, shared language, connectors, Notion, agent chaining, and deep research.

Open a new conversation with Benefit. Go to each learning-chapter page on the site, click the "copy to Notion" button, and paste the chapter into the conversation. At the top of the conversation, before the first chapter, paste this prompt:

```
I'm going to paste you learning chapters from the course, one at a time. For each chapter: identify which unit and which session it belongs to, find its right place in the notebook (Benefits → the session → the unit), and add the matching STUDIO tag. Before you write anything to Notion, first show me where the chapter goes and which tag you gave it, and wait for my approval. Let's start. First chapter:
```

Under the prompt, paste the first chapter. After Benefit shows you where it's filing and you've approved, type "next chapter:" and paste the next one, and so on until all eight chapters are in.

**Why this:** this is what turns Benefit from "knows today's lesson" into "knows the whole course." And filing by the structure is exactly what lets it later pull the right page for you, without searching blind.

## Task four: a real decision from this week

Pick one real decision you're sitting with this week. A tool to choose, an email that's hard to word, a small project you don't know where to grab. Hand it to Benefit, ask for STUDIO advice, and at the end save the insight to Notion:

```
I have a real decision from this week: [describe it in two lines]. I want STUDIO advice: where to start, which tool to use at each stage, and the first thing I should do right now. At the end, sum up the key insight and save it to Notion on my insights page.
```

**Why this:** this is the whole point of Benefit. Not to store the course, but to sit next to you on a real decision, and to keep what it helped you see, so that next time it's already there.

## A few things worth remembering

The memory is always in your hands: you can open Notion and fix a name or a date yourself. It's better to let Benefit file, but when you need to, you can. If a write doesn't go through, check that Notion is connected with write permission, and approve the write tool when it asks. And if you get stuck at any step, try again, and we'll sort it out together at the session.
